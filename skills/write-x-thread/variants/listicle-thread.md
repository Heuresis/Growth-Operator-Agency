# X Thread Variant — Listicle Thread

> **Signature move.** A specific numbered list — "7 tools," "5 mistakes," "12 frameworks," "9 emails" — each item given one dedicated tweet with a named concept, a short explanation, and a specific example. The thread sells compressed value — the reader gets a framework they can save, reference, and act on.

## When to Use

- Pillar is How-To, Toolkit, or Curation.
- Conversion stage is Awareness or Consideration.
- Creator has a specific, named list that isn't padded.
- List items are genuinely distinct — not variations of the same idea.

## When NOT to Use

- List would be padded to hit a round number. If the creator has 5 strong items and 3 weak ones, ship the 5-item thread.
- List is not actually a list — it's a narrative or framework. Use Educational Thread instead.
- Items require 2+ tweets each to explain. The list should fit the "one tweet per item" constraint; if it doesn't, the content is too complex for Listicle format.
- Topic is so broad the list feels arbitrary. Lists require specificity.

## Opener (Hook Tweet)

Listicle hooks lead with the number.

Opener patterns:

1. **Number + specific category.** "7 frameworks every creator charging $10K+ should have memorized:"
2. **Number + outcome.** "5 mistakes that kept me flat at $8K/mo for 18 months:"
3. **Number + timeframe.** "12 emails I write before every launch (in this order):"
4. **Number + contrarian claim.** "9 things most productivity advice gets wrong. I made all 9 of them:"
5. **Number + authority signal.** "4 tools I use daily after testing 50+ over 7 years:"

Hook tweet rules:

- Number in the first 5 words.
- Specific category named.
- Authority signal if available — "after testing 50+," "from $X to $Y," "in 400 engagements."
- 120-180 characters — numbered hooks should be tighter than educational.

## Body Structure

### The Clean Numbered List (default)

Each tweet is one item:

```
[N]/ [Named Item]

[One-sentence explanation]

[Specific example or application]
```

Numbers go at the start so readers can visually track progression. "1/" "2/" "3/" format works cleanly.

### The Countdown List (for ranked lists)

- Start with N at the top: "12/ [least important or most common misconception]"
- Work down to "1/" as the most important / biggest impact.
- Tweet 1/ is the punchline — the tweet that gets bookmarked.

Works for myth-busting lists ("10 things most creators get wrong, ranked by cost") or tool rankings.

### The Ascending List (for sequence lists)

- Start with "1/" as the first step.
- "N/" is the final step or outcome.
- Works for sequence-based lists — "9 emails I write before launch, in this order."

### The Categorized List (for 7+ item lists)

- Break the list into 2-3 categories with a quick category header.
- "CATEGORY A (items 1-4):" then items 1-4.
- "CATEGORY B (items 5-7):" then items 5-7.
- Helps readers organize the volume.

## Bookmark-Bait Patterns

Listicles are among the highest-bookmarked thread types. Optimize aggressively.

- **Summary tweet.** A single tweet mid-thread (usually after item 3 or 4) that lists all N items in one scroll. This is the "save the list" tweet.
- **Named items.** Each item has a memorable name, not just a description. "The 24-hour rule," "The isomorphic case study," "The founder-as-closer window" — names stick.
- **Parallel formatting.** Each item tweet has the same structure. Readers can skim-save.
- **Clean line breaks.** White space between item name, explanation, and example.

## Re-Engage Structure

- **Reply-to-thread (T+24-48h)** — the creator quote-tweets their own thread with "the item that surprised most people was #[N]."
- **Quote-tweet (T+48-72h)** — second quote with one additional item not in the original list ("bonus 13th: [item]").
- **Re-post pattern** — well-structured listicles re-thread every 4-6 months with updated items. This is legitimate re-use, not recycling.

## Retweet Invitation

- The summary tweet (mid-thread) is the most-retweeted tweet in listicle threads.
- Item 1/ (in countdown format) or the final item (in ascending) is the second-most retweeted.
- Final CTA tweet occasionally gets retweeted if the call is strong ("save this list — I update it every quarter").

## CTA / Close

Single action.

- "Save this list. I update it every [timeframe]."
- "Follow @[handle] for more lists like this."
- "DM 'LIST' and I'll send the full PDF version." (Capture mechanism.)
- "Which item surprised you most? Reply below." (Engagement prompt.)

## Character Limits

- 280 max per tweet.
- 150-250 ideal for item tweets — more compressed than educational.
- 100-180 for the hook.
- Summary tweet can push to 280.

## Voice-Match Requirements

- 3+ `phrases_to_use` across the thread.
- Zero items from `spec/BANNED-VOCABULARY.md`.
- Listicle voice is direct and compressed. Less analytical than Educational.
- Named items use the creator's natural phrasing if they have one.

## Specificity Floor

- Every item has a specific name.
- Every explanation has a specific detail — number, example, situation.
- The list total is honest. No padding to a round number if the content doesn't support it.

## Compliance Check

- Banned vocabulary swept.
- No income guarantees.
- Claims specific enough to defend.
- Named items don't infringe on copyrighted framework names.

## Common Failure Modes

- **Padding to a round number.** 7 becomes 10 with three weak additions. Readers notice.
- **Items blend together.** 5 and 6 are variations of the same idea. Merge or cut.
- **No named items.** Each item is just a description. "Do X" instead of "The X Principle." Named items bookmark better.
- **Missing summary tweet.** Readers can't see the whole list at a glance. Summary tweet is the save-the-thread tweet.
- **Countdown confusion.** Starting with 10/ without signaling countdown — readers expect ascending.
- **Over-long items.** Each item takes 280 characters when 180 would do. Readers lose the skim rhythm.

## Hand-off Checklist

- Hook tweet leads with the number.
- N body tweets, one per item, 150-250 characters each.
- Summary tweet included (especially for 7+ item lists).
- Final CTA tweet — single action.
- Numbers format-consistent ("1/", "2/", not "1." then "2:").
- Zero banned vocabulary.
- Voice-match verified.
- Reply + quote-tweet scheduled.
