# YouTube Variant — How-To Tutorial (10-20 min)

> **Signature move.** A specific task or workflow taught step-by-step, with the end result visible at the start and the steps recreated precisely over 10-20 minutes. The viewer finishes able to do the thing. The video sells search-optimized evergreen authority — it ranks in YouTube search and attracts viewers who arrived with intent.

## When to Use

- Task is specific, reproducible, and takes 10-60 minutes of actual doing to complete.
- Pillar is How-To, Tool-Tutorial, or Workflow-Teaching.
- Conversion stage is Awareness — tutorials attract search traffic, the best top-of-funnel source for evergreen audience growth.
- The creator is the operator on this specific task and has done it repeatedly.

## When NOT to Use

- Task is conceptual (no clear "do this, then this" structure). Use Educational Deep-Dive instead.
- Task has only one viable way to do it, and that way is fully documented in official docs elsewhere. Tutorial adds no value if it's just re-narrating a manual.
- Task requires proprietary data that can't be shown on screen without redaction that breaks the flow.
- Task involves live judgment calls that can't be prescripted into steps.

## Title Formula

Pattern: `How to [Specific Outcome] ([Tool or Method]) — [Timeframe or Qualifier]`

Examples (archetypal):

1. "How to Build a Book-a-Call Funnel in Framer (Under 30 Minutes, No Designer)"
2. "How to Write a Cold Email Sequence That Gets 40% Reply Rate (5 Emails, 2 Minutes)"
3. "How to Set Up a Notion Revenue Dashboard (The 4 Metrics That Actually Matter)"

Title rules:

- 60-70 characters.
- Starts with "How to" — search-aligned.
- Names the tool if the tool is central ("in Framer," "in Notion").
- Names a timeframe or qualifier that sets expectation ("under 30 minutes," "no designer," "5 emails").

## Thumbnail SOP

Tutorial thumbnails visualize the end result.

Options:

- **Result-visible thumbnail.** The finished artifact visible. "Before" on left (blank state), "After" on right (completed artifact). Minimal face.
- **Tool-logo thumbnail.** Tool logo + the specific outcome. Creator face in corner.
- **Screen-capture thumbnail.** A screenshot of the finished thing, with an arrow and a "DO THIS" annotation.

Text overlay: the specific outcome or timeframe ("30 MIN," "NO CODE," "40% REPLY").

## Script Structure

### Intro (0:00 - 1:00)

- **0:00 - 0:10 — End-Result Reveal.** Show the finished artifact first. "This is what we're building. Here's what it does."
- **0:10 - 0:25 — Who This Tutorial Is For.** Prerequisites, skill level, what they need ready.
- **0:25 - 0:45 — Time Estimate + Tools.** "This takes 25 minutes if you follow along. You need: [tool 1], [tool 2]."
- **0:45 - 1:00 — Roadmap.** The step list — 4-8 named steps that the viewer will complete.

### Setup (1:00 - 2:30)

- Starting state visible.
- Accounts / tools open and ready.
- Any pre-reqs named and handled.
- A note on what the viewer can skip if they've done prior setup.

### Step 1 (2:30 - 4:30)

Each step has:

- **Do:** The specific action.
- **Why:** Why this step matters.
- **Common mistake:** What people get wrong here.
- **Expected result:** What their screen should look like.

### Step 2 (4:30 - 6:30)

Same structure.

### Step 3 (6:30 - 9:00)

Same structure.

### Step 4 (9:00 - 11:30)

Same structure.

### Step 5 (11:30 - 14:00)

Same structure.

### Step 6 / Final (14:00 - 16:00)

Same structure, plus the final-state check.

### Troubleshoot (16:00 - 18:00)

- Top 3 things that break.
- How to diagnose each.
- How to fix each.

### Polish / Advanced (18:00 - 19:00)

- 1-2 optional refinements for viewers who want to go further.

### Outro / Binge Loop (19:00 - 20:00)

- "Now you have X. Here's how to make it better in the next tutorial."
- Next video tease (logical next tutorial in the workflow).
- Subscribe ask.
- End-screen.

## CTA Placement

- Soft CTA at minute 2-3: "Grab the template in the description." (The template is the search-to-email capture mechanism.)
- No paid offer CTA. Tutorials are top-of-funnel.
- Comment prompt at minute 16 (troubleshoot chapter): "What broke for you? Drop it in comments, I'll help."
- End-screen: next tutorial in the workflow series.

## Retention Curve

Target: AVD above 55% for 10-20 min tutorials (higher than Deep-Dive — tutorial viewers have intent).

Defense points:

- **Minute 0:00 - 0:10** — result-first reveal earns the stay. If viewers don't see the end state in the first 10 seconds, they bounce to search result #2.
- **Minute 2:30** — first real step. Viewers who stay through setup are committed.
- **Minute 11:30** — halfway point. Some viewers who came for the first 3 steps leave. That's fine.
- **Minute 16:00** — troubleshoot chapter. Surprisingly high-retention chapter because viewers who got stuck in their own attempt come back for this.
- **Minute 19:00** — outro. Retention usually drops here. That's expected.

## End-Screen Strategy

- Next video: the logical next tutorial in the workflow series.
- Playlist: the tutorial series for this topic.
- Subscribe element.

Tutorial end-screens can include a lead-magnet link — the template file the viewer just built is a natural email-capture.

## Loop Management

Tutorials use fewer curiosity loops than Deep-Dive — the "loop" is the result itself.

- Loop 1 opened at 0:10 (the end-state) — progressively closed as each step builds toward it.
- Loop 2 opened at 2:00 ("the step most tutorials skip") — closed at the relevant step.
- Loop 3 opened at 11:00 ("the thing that looks hard but is actually easy") — closed at the relevant step.

## Voice-Match Requirements

- 3+ `phrases_to_use` per 5-minute segment.
- Zero items from `spec/BANNED-VOCABULARY.md`.
- Tutorial voice is instructional and clear, not performative. Short sentences. Clear verbs. Avoid "let's" openers — go straight to the action.

## Specificity Floor

- Every step names the exact button, menu, field, or element on screen.
- Every instruction includes the expected outcome — "Click here. You should see X."
- Every common mistake is specific — not "don't mess this up" but "don't leave this field blank; if you do, X happens."

## Script Word Count

- 10-min: ~1,400-1,500 words
- 15-min: ~2,100-2,250 words
- 20-min: ~2,800-3,000 words

## Production Notes

- Primary mode: screen recording with voice-over.
- Creator face-cam inset for 20-30% of runtime — at the intro, at major decision moments, at troubleshoot, and at outro.
- Cursor highlight / zoom annotations on every click — viewers cannot follow small cursor movements at 1080p mobile.
- Real-time or 1.5x speed — do not time-lapse. Tutorial viewers want to follow along at pace.
- Keyboard shortcuts displayed as text overlays when used.
- Audio: clean mic, no room noise. Tutorial is often listened to while the viewer works.

```

[2-3 paragraphs — what this tutorial covers, what you'll have at the end, prerequisites]

[Hashtags — tool names, outcome keywords]
```

## Common Failure Modes

- **No end-result at intro.** Viewer doesn't know what they're building until minute 3. By then they've bounced. Show the end-state in the first 10 seconds.
- **Narrated manual.** Tutorial just reads the tool's docs out loud. Add the "why" and "common mistake" to every step, or the tutorial is redundant with the official docs.
- **Skipped steps.** Creator skips steps they assume are obvious. Viewers who don't know the obvious ones get stuck. Include every step — if viewers want to skip, they can skip.
- **No troubleshoot chapter.** Viewers who hit problems leave. Troubleshoot chapter is where they return to the video and finish.
- **Over-polished editing.** Jump cuts every 2 seconds. Viewers lose the thread. Tutorials need steady pacing, not reel-style cuts.

## Worked Example — Book-a-Call Funnel in Framer

### Title
"How to Build a Book-a-Call Funnel in Framer (Under 30 Minutes, No Designer)"

### Thumbnail
- Left: blank Framer project
- Right: finished landing page with visible calendar embed
- Arrow between: "30 MIN"
- Creator face, lower-right corner

### Script Outline (abbreviated)

```
INTRO (0:00 - 1:00)

0:00 - 0:10 END-RESULT REVEAL
[Screen: finished funnel.]
VO: "This is what we're building. One landing page with a three-proof hero and a calendar embed. Viewers who land on this book a call at ten to fifteen percent when traffic is qualified."

0:10 - 0:25 WHO THIS IS FOR
VO: "This is for founders or consultants selling offers between three and thirty thousand. You need: a Framer account (free works), a calendar tool with a shareable embed, and three case studies you can name specifically."

0:25 - 0:45 TIME + TOOLS
VO: "Thirty minutes if you follow along. Here are the tools on screen — Framer on the left, your calendar tool on the right."

0:45 - 1:00 ROADMAP
VO: "Seven steps. Start a project. Build the hero. Build three proof rows. Add the calendar embed. Set up the mobile view. Add tracking. Ship."

SETUP (1:00 - 2:30)
Framer opened, blank project ready.

STEP 1 — Start the Project (2:30 - 4:30)
Do: create new Framer project, select the 'landing' template.
Why: landing template starts with the right constraints — single-page, mobile-first.
Common mistake: starting from 'website' template, which assumes multi-page and adds friction.
Expected result: single-page layout, hero + section divider on screen.

STEP 2 — Build the Hero (4:30 - 7:00)
Do: headline, sub-head, CTA button.
Specific copy structure on screen.
Why: hero does 60% of the conversion work.
Common mistake: two-line headline when one line is stronger.
Expected result: hero section looks like [screenshot].

STEP 3 — Three Proof Rows (7:00 - 10:30)
Do: three horizontal rows under hero, each with a case study card.
Why: one proof is anecdote, three is pattern. Pick three that are isomorphic to the viewer's starting stage.
Common mistake: using your biggest case study. Better: your most relatable.
Expected result: three cards, each ~120px tall, matching the hero design.

STEP 4 — Calendar Embed (10:30 - 14:00)
Do: add the calendar embed block, paste the URL.
Why: the embed must live on the page — no 'click here to book' button. The calendar is the conversion.
Common mistake: embed not set to 'seamless' mode; viewers see a scroll-jail.
Expected result: calendar loads natively in-page.

STEP 5 — Mobile View (14:00 - 18:00)
Do: switch to mobile breakpoint, adjust padding and font sizes.
Why: 70% of traffic is mobile. Desktop-only looks cheap to mobile visitors.
Common mistake: leaving the proof cards in two columns on mobile. Single column on mobile.
Expected result: mobile view matches the desktop aesthetically but fits phone frame.

STEP 6 — Tracking (18:00 - 21:00)
Do: add Meta pixel, Google Analytics, calendar-book conversion event.
Why: without tracking, you cannot diagnose which step leaks.
Common mistake: pixel installed but event not firing on calendar-booking.
Expected result: pixel test shows page view + custom event when calendar-book action fires.

STEP 7 — Ship (21:00 - 23:00)
Do: publish, verify live URL, test the calendar booking end-to-end.
Why: publish is the moment. Test a live booking before you send paid traffic.
Common mistake: publishing and not testing. Leads slip through.
Expected result: live URL, test booking appears in calendar.

TROUBLESHOOT (23:00 - 26:00)
- Calendar doesn't load: usually the URL is the admin link, not the embed link.
- Mobile looks broken: usually the proof cards not converted to single column.
- Pixel not firing: Meta pixel helper extension on Chrome to verify.

ADVANCED (26:00 - 28:00)
- Optional: A/B test the headline with a simple rotator.
- Optional: add a sticky mobile CTA button.

OUTRO (28:00 - 30:00)
"Now you have a book-a-call funnel. Next video — the email sequence that follows a no-show."
End-screen.
```

### Retention Defenses

- 0:10 end-result hook earns the 30-minute commitment.
- 4:30 first real step is the transition from setup to doing.
- 14:00 mobile view chapter is a texture moment that separates good tutorials from great ones.
- 23:00 troubleshoot chapter catches viewers who got stuck in their own attempt.

## Compliance Check

- Banned vocabulary swept.
- No fabricated results ("this gets 15%" — is that claim backed by your own data?).
- If tools are named, they are actually used.
- If templates are shared, they work.
- FTC disclosure if any tool is affiliate-linked.

## Hand-off

Tutorial deliverables:

- Full script (1,400-3,000 words).
- Title + 2 variants.
- Thumbnail concept + 1 variant.
- Chapter markers.
- Full screen-recording storyboard.
- Callout / zoom annotation list with timestamps.
- Template file attached (the viewer's capture mechanism).
- Description with SEO copy.
- Compliance check signed off.
