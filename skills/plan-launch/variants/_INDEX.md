# skills/plan-launch/variants/

Per-type playbooks for the 5 launch archetypes `/plan-launch` can produce. Each file covers the full 5-phase SOP (Pre-Launch / Launch Day / Momentum / Follow-Up / Analyze), communication calendar, asset checklist, and common failure modes.

## The 5 launch types

| File | Archetype | When it wins |
|---|---|---|
| [live-cohort-launch.md](live-cohort-launch.md) | Live cohort launch | Scheduled start date + real seat cap + community delivery |
| [evergreen-automation-launch.md](evergreen-automation-launch.md) | Evergreen automation | Always-on funnel replacing one-time launches; scaling via paid spend |

## How the launch types relate to the funnel archetypes

Launches and funnels aren't the same thing. A **funnel** is the mechanism by which a prospect reaches an offer (ongoing). A **launch** is a time-boxed campaign that uses one or more funnels to concentrate revenue in a specific window.

Typical pairings:

- `live-cohort-launch` ↔ `challenge-funnel` OR `webinar-funnel` OR `multi-video-prelaunch-funnel`
- `evergreen-automation-launch` ↔ `vsl-funnel` OR `tripwire-funnel` OR `webinar-funnel` (evergreen)
- `flash-sale-launch` ↔ existing list + brief reopening of an existing offer
- `multi-video-prelaunch-runway` ↔ `multi-video-prelaunch-funnel` (same mechanism, different framing — runway is the education arc, funnel is the conversion pipe)
- `jv-stack-launch` ↔ any of the above, but scaled by N partners

## The 5-phase SOP (shared across all types)

Every launch variant follows the same 5-phase sequence; the specifics change per archetype.

1. **Pre-Launch** (2–12 weeks before) — asset production, list warming, partner mobilization, tech QA
2. **Launch Day** (cart opens) — coordinated promo across channels, first-day push, early-bird capture
3. **Momentum** (days 2–N of cart window) — daily push, objection addressing, proof-stacking, community scaling
4. **Follow-Up** (last 24–48 hours) — urgency escalation, bonus expiration, final-hours push
5. **Analyze** (within 2 weeks after close) — `/launch-report` full debrief + next-launch improvements
