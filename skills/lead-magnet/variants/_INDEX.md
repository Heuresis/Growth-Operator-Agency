## Variant Selection Decision Tree

```
IF creator needs fast-consumption utility:
    → checklist or cheatsheet

ELIF creator needs mental-model teaching in one session:
    → pdf-guide

ELIF creator needs tactical reference that gets re-used:
    → swipe-file

ELIF creator needs serialized belief installation across days:
    → minicourse

ELIF creator needs archetype segmentation or cold-traffic conversion:
    → quiz

ELIF ICP thinks in numbers and ROI:
    → calculator

ELIF creator needs high-intent prospect filter and deep authority:
    → live-training

ELIF ICP thinks in tools and workflows:
    → custom-artifact

ELSE: default to pdf-guide (broadest applicability)
```

## Upstream and Downstream Routing

Every variant feeds into an /email-sequence:

- **welcome-sequence** always follows delivery (default downstream)
- **application-sequence** if reader clicks bridge-to-offer CTA that points to application
- **webinar-promotion-sequence** if variant is live-training
- **nurture-sequence** for readers who consume but do not click

Upstream:
- **/landing-page** builds the opt-in page
- **/ad-creative** drives traffic
- **/content-calendar** queues organic-to-magnet bridges

## Universal Rules (apply to every variant)

Every variant must:
- Deliver under 120 seconds of opt-in
- Bridge to a defined core offer or next-step asset
- Name the creator's mechanism
- Use 2 or more `phrases_to_use` (3 or more for text-heavy formats)
- Pass the full banned-vocabulary sweep
- Include attribution and any required compliance disclaimers
- Permission-clear every case study, testimonial, or third-party asset
