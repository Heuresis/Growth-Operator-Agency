# Custom Artifact Playbook

## Role In the Funnel

A custom artifact is a functional asset the reader actually uses in their work. Templates get duplicated. Spreadsheets get filled in. Notion workspaces get adopted. Unlike PDFs which are read, custom artifacts are operated.

Three jobs:

1. Deliver a tool the reader uses repeatedly, not a document they read once.
2. Embed the creator's mechanism into the tool's structure. The reader adopts the mechanism by using the template.
3. Create a persistent brand presence in the reader's own workspace. Every time they open the template, the creator's name and mechanism are visible.

A custom artifact is not a PDF guide and is not a cheat sheet. Guides teach. Cheat sheets reference. Custom artifacts operate. The reader's relationship to the artifact is active, not passive.

## Gate-in Conditions

Ship a custom artifact only if:

- Compartment 2 (Audience) at 65 percent; artifact format must match ICP's tool stack
- Compartment 3 (Offer) at 50 percent
- Creator's mechanism can be embodied in a functional tool structure
- Host platform selected based on ICP tool familiarity (Notion for creators, Airtable for operators, Google Sheets for widest compatibility, Figma for designers, custom for developers)
- Legal review complete for licensing and redistribution terms
- Template is tested by 3 to 5 beta users before public release

## Custom Artifact Formats

Pick one format per lead magnet:

1. **Spreadsheet template:** Google Sheets or Excel with pre-built formulas, tabs, and sample data. Examples: revenue forecaster, content calendar, client tracker, goal modeler.
2. **Notion template:** multi-page Notion workspace with databases, linked views, and embedded documentation. Examples: content OS template, launch management workspace, project tracker.
3. **Airtable template:** base with pre-built views, automations, and sample records. Examples: customer database, pipeline tracker, campaign dashboard.
4. **Miro or Figma template:** visual canvas with pre-built frames, diagrams, or workflow maps. Examples: user journey map, funnel diagram, strategy canvas.
5. **Prompt pack:** curated collection of LLM prompts for a specific task, with usage instructions. Examples: content ideation prompt pack, sales-call analysis prompts, research synthesis prompts.
6. **Code snippet library:** functional code samples for a specific task with docs. Examples: copy-paste HTML components, tracking setup scripts, automation workflows.
7. **Workbook:** interactive PDF or Google Doc with fill-in fields, questions, and scoring rubrics. Examples: positioning workbook, offer-design workbook, ICP builder.
8. **Slide template pack:** pre-designed slides for pitches, proposals, or case studies. Examples: sales-deck template, workshop slide pack, case-study presentation.

## Artifact Design Rules

Each artifact has:

- **A "How to use this" intro.** First page, first sheet, or first screen. Explains what the artifact is, what it produces, and how to duplicate it for the reader's own use.
- **A demo row, demo entry, or demo section.** Shows the reader what filled-in looks like so they can pattern-match their own use.
- **The creator's mechanism embedded in the structure.** Section names, sheet titles, prompt categories, or database field names reflect the mechanism's sub-steps.
- **A "next step" link or CTA.** Subtle but present. Pointing to the core offer or a deeper resource.
- **Attribution and licensing.** Creator credit, original URL, and clear licensing (personal use, commercial use, or redistribution terms).

## Opt-in Page Copy Structure

- Hook headline: template title with format named ("The Template," "The Prompt Pack," "The Workspace").
- Subhead: "Built for [ICP Role]. Duplicate and use in [N] minutes."
- Visual preview: screenshot of the artifact in use, showing the filled-in demo content.
- Three to five bullets naming what the artifact includes (sheets, databases, prompts, frames).
- Platform requirement note: "You'll need [Notion / Airtable / Google Sheets / Figma] access. It's free."
- Single CTA button: "Send me the template."
- Attribution and licensing fine print.

Opt-in page under 400 words. Visual preview is the sales copy.

## Delivery Automation

- Opt-in fires on submit.
- ESP tags: `lm-template-[slug]`, `welcome-active`.
- Delivery email under 120 seconds with:
  - Link to duplicate the template (Notion share link, Airtable base share, Google Sheets copy link, Figma duplicate link).
  - One-line instruction for duplicating to the reader's own workspace.
  - Link to a short video walkthrough if the artifact is complex.
- Welcome sequence begins immediately after delivery.

## Host Platform Choice

| ICP Stack | Preferred Host |
|---|---|
| Creator solo, Notion-native | Notion |
| Small team, operations-heavy | Airtable |
| Broad audience, low tool commitment | Google Sheets or Google Doc |
| Designers, visual thinkers | Figma or Miro |
| Developers, technical | GitHub repo with README |
| Marketers, content ops | Notion or Airtable |

Pick the host that matches the ICP's existing tool stack. A Notion template delivered to a non-Notion ICP has low activation rate regardless of quality.

| Metric | Healthy Range | If Below | If Above |
|---|---|---|---|
| Opt-in rate on warm traffic | 40 to 60 percent | Format mismatch with ICP stack | Fine |
| Opt-in rate on cold traffic | 25 to 35 percent | Title vague or host platform wrong | Fine |
| Delivery email open rate | 75 to 90 percent | Sender issue | Fine |

## Common Failure Modes

1. Host platform does not match ICP stack. Template gets opened but not used. Fix: match platform to ICP's existing tools.
2. No "how to use this" intro. Reader opens template and does not know what to do. Fix: first sheet, first page, or first screen explains the artifact end-to-end.
3. Demo content missing. Reader cannot see what filled-in looks like. Activation drops. Fix: always include a demo row, entry, or section.
4. Mechanism is not embedded in the template structure. Template is useful but brand-neutral. Fix: section names, field names, and prompt categories reflect the mechanism.
5. No attribution. Template spreads without credit. Fix: creator credit visible on every page, sheet, or screen of the artifact.
6. Licensing unclear. Readers hesitant to use in client work. Fix: explicit licensing terms stated in the artifact and the delivery email.
7. Template is too complex to use without a tutorial. Activation drops below 30 percent. Fix: include a short video walkthrough plus simplified quick-start section.

## Voice-Match and Compliance

- 2 or more `phrases_to_use` on delivery email, intro, and any copy within the artifact.
- Banned-vocabulary sweep clean on title, delivery copy, artifact intro, and walkthrough video script.
- Licensing terms compliance-reviewed.
- Attribution consistent across every page or screen.
- No third-party content included without permission.
- Privacy compliance for any embedded tracking (Notion view tracking, Airtable usage analytics) disclosed.

## Next Step Routing

After delivery, welcome sequence begins. Bridge CTA and intro block point to:

- High-ticket offer: application or calendar
- Mid-ticket offer: VSL or landing page
- Community funnel: community join page
- Course or program offer: course enrollment page

Custom artifacts frequently lead to secondary opt-ins. A reader who uses a Notion template often opts in for a follow-up training, swipe file, or course.
