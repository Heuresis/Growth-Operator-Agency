# Reel Variant — Speed Build

> **Signature move.** "I built X in Y minutes." The reel compresses a real piece of work — a landing page, a cold-email sequence, a dashboard, an offer deck, a funnel, a lead magnet, a contract — into a time-lapse or step-compression that ends with a visible artifact. Tension is "can that actually work." Resolution is the working artifact on screen. The reel sells capability, not a product.

## When to Use

- The creator ships a lot of real artifacts and wants to convert creator-level capability into authority signals.
- Pillar is How-To, Capability Showcase, or Tool Demonstration.
- Conversion stage is Awareness or early Consideration — building recognition of the creator as the person who can make a specific kind of thing.
- An actual working artifact exists at the end. Speed-build without a working end-product is a lie.

## When NOT to Use

- The "build" is aspirational or mocked — the viewer can tell, and the creator loses trust.
- Offer depends on deep expertise the viewer cannot observe in a time-lapse (therapy, strategy, retention).
- The artifact is sensitive / private / under NDA — redaction that blocks the payoff is worse than not posting.
- The creator is not on-camera and the artifact is not compelling enough to carry the reel without commentary.

## Structure — 60-Second Grid

| Time | Beat | Purpose |
|---|---|---|
| 0:00-0:03 | Promise Hook | "I'm going to build X in Y minutes" — state the artifact and the time box |
| 0:03-0:08 | Setup | Blank screen / empty doc / starting state visible |
| 0:08-0:45 | Build Sequence | Time-lapse or step-compressed showing the build, with text-overlay commentary |
| 0:45-0:55 | Reveal | Working artifact shown at full scale |
| 0:55-0:58 | Takeaway Rule | What the viewer now knows about how to do this |
| 0:58-1:00 | CTA or Loop | "Follow for more" or loop to promise |

## Hook Patterns (5 specific openers)

1. **Time-box + specific artifact.** "I'm going to build a $10K landing page in 4 minutes." The time box and the outcome are both specific.
2. **Before-state + after-promise.** "This is a blank Notion page. In 60 seconds, it's a full onboarding SOP." The transformation is the hook.
3. **Cost-compression.** "My agency charges $3,500 for this. I'm going to build it live in 3 minutes." The tension is cost vs. time.
4. **Stack-reveal.** "Three tools, four minutes, one working funnel. Watch." The viewer commits to see three named things compressed.
5. **Constraint-promise.** "No code, no designer, no money. I'm going to ship a landing page in 2 minutes." Scarcity of resources is the hook.

## Body Patterns

### A. Time-Lapse Compression (default — 15-45x real speed)

- The entire build is actually done. Footage runs at 15-45x speed so a 15-minute task compresses to 30-60 seconds of reel body.
- Creator voice-over narrates at real speed; the footage speeds around the voice.
- Text overlays appear at 3-5 second intervals to name what's happening ("Now adding the headline," "Now importing the proof asset").

### B. Step-Compression (for multi-tool builds)

- Break the build into 3-5 steps. Each step is a 6-10 second segment at real speed or 2x speed.
- Transition between steps with a hard cut + text overlay naming the step.
- Useful when the build spans tools (e.g., Notion to Figma to Framer) because time-lapse across tools looks chaotic.

### C. Real-Time with Callouts (for fast tools)

- If the build genuinely fits in 45 seconds at real speed (e.g., a simple Typeform, a one-liner email template), play the footage at 1x.
- Use callouts to name the invisible decisions ("I'm choosing this field type because of Y," "This setting is the one most people miss").

## CTA and Loop Patterns

- "Follow for more builds like this." — default for Awareness-stage audiences.
- "Comment [specific word] and I'll send the template file." — the template is the hook into email capture, the reel is the lead magnet trigger.
- Loop pattern — last frame shows the finished artifact; the first frame shows the blank starting state. Viewers who replay see the transformation reverse-then-forward, which reads as deliberate craft, not accident.

## Voice-Match Requirements

- Three-plus `phrases_to_use` from Compartment 1 in the voice-over.
- Zero items from `spec/BANNED-VOCABULARY.md`.
- The voice-over must match the time-lapse pacing — short sentences, one thought per breath, because each sentence overlays 5-10 seconds of compressed footage.

## Specificity Floor

Hook Specificity ≥ 8. The floor:

- The time box is a specific integer number of minutes or seconds, not "fast" or "quickly."
- The artifact is named specifically, not "a funnel" but "a book-a-call funnel for coaches charging $3,000-plus."
- The tool is named if it is central, not "a tool" but "Framer" or "Notion" or "Superhuman."

## Visual Brief (scene-by-scene shot list)

- 0:00-0:03 — Creator on-camera or voice-over over blank screen. Text overlay states the promise.
- 0:03-0:08 — Blank starting state visible. Optional: creator's hands entering the frame to begin.
- 0:08-0:45 — Screen recording OR top-down build footage, time-lapsed. Overlay text at 3-5 second intervals naming what's happening.
- 0:45-0:55 — Cut to the finished artifact at full resolution. Hold for 8-10 continuous seconds so the viewer can read and register.
- 0:55-0:58 — Creator on-camera delivering the takeaway rule.
- 0:58-1:00 — Text-overlay CTA or re-state of the opening promise.

## Platform Adaptations

- Instagram Reels: vertical 9:16 screen recording. Framer / Figma / Notion are natively vertical or tolerate cropping. Caption opens with the promise. Hashtags are tool-specific.
- TikTok: the first 0.5 seconds is a pattern interrupt — the artifact flashes for one frame before cutting to the blank starting state. This is TikTok-native "spoiler hook" behavior.
- YouTube Shorts: title is the promise literal. Description includes a link to a longer YouTube teardown of the same build.

## Compliance Check

- Banned vocabulary swept.
- The build is actually done — no fake screen recordings, no AI-generated mock-ups presented as built artifacts.
- If tools are named, the creator actually used those tools.
- If a third party's brand shows in the build (e.g., the creator builds a landing page for "Acme Co" as a demo), get permission or redact the brand.

## Common Failure Modes

- **The promise and the artifact mismatch.** Hook promises a "landing page," body builds a homepage. Viewers feel bait-and-switch.
- **The time-lapse is too fast to follow.** If the viewer can't read what's happening in the overlay text, they disengage. Keep time-lapse at 15-25x, not 60x.
- **The reveal isn't legible.** If the finished artifact is blurry or too small on a phone screen, the payoff dies. Zoom into the readable sections of the artifact for the reveal.
- **No takeaway rule.** The reel shows capability but gives the viewer nothing to take away. Every Speed Build should end with "the rule the build taught."
- **Overclaiming the build quality.** "A $10K landing page" should actually look like a $10K landing page. If the build is basic, claim a basic build ("a 2-minute landing page that converts at 40%").

## Worked Example 1 — Landing Page Build

```
0:00-0:03 | Creator on-camera, laptop visible.
TEXT: "Book-a-call landing page. 3 minutes. No designer."
VO: "I'm going to build a book-a-call landing page in three minutes. No designer."

0:03-0:08 | Blank Framer project visible.
VO: "Starting from scratch. Framer."

0:08-0:20 | Time-lapse screen recording — creator builds the hero section.
TEXT (rotating): "Headline → Sub-head → Proof row → CTA"
VO: "Four parts to the hero. The headline names the outcome. The sub-head names who it's for."

0:20-0:32 | Time-lapse — creator builds the three social proof rows.
TEXT: "Three proof assets — not one"
VO: "Three case studies, not one. Each one names a different starting stage."

0:32-0:40 | Time-lapse — creator wires the calendar embed and the CTA.
TEXT: "Calendar embed + one CTA"
VO: "One call-to-action. One calendar. No second offer."

0:40-0:55 | Cut to the finished landing page, full-screen scroll-through.
VO: "Done. Three minutes. Here's the page."

0:55-0:58 | Creator on-camera.
VO: "The rule: a landing page is a hero, three proofs, one CTA. Everything else is decoration."

0:58-1:00 | Text-overlay CTA.
TEXT: "Comment LANDING for the Framer template"
```

## Re-Engage Pattern

- T+24h pinned comment: the template file, a link to a walkthrough, or an upgraded version the viewer can rebuild.
- T+48h quote-repost: creator posts a follow-up reel showing the artifact "in the wild" — a real client using it.
- T+7d long-form: a YouTube video breaking down the same build in 10-15 minutes, with the reel as the teaser.

## Hand-off to Editor

- Screen recordings provided at native resolution.
- Time-lapse speed specified per segment.
- Text overlays timed to each 3-5 second interval.
- Music is minimal — tool-UI sounds kept audible.
- Last-frame / first-frame match for the loop.
