# Competitor Reference

> **Purpose:** per-creator competitor teardowns produced by `/competitor-intel`.
>
> **Status:** populated at runtime. Empty at the template level.

When an operation runs `/competitor-intel`, outputs land here at `reference/competitors/<competitor-slug>.md`. Each file is a battlecard with: positioning, offer, pricing, proof stack, mechanism, gaps, whitespace.

- `../../skills/competitor-intel/SKILL.md` — produces these files
- `../../skills/competitor-intel/examples/` — sample battlecards
