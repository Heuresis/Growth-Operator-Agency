# high-value stacked offer

> **Source:** the acquisition economist — a high-value offer canon (Chapters 4-10)
> **Status:** Companion to the offer architect's 7-step. Primary Value Equation operator.

## The Definition

> *"A high-value stacked offer is an offer you present to the marketplace that cannot be compared to any other product or service available, combining an attractive promise with an unmatchable value equation, incredible bonuses, unbeatable guarantees, and a supportive payment plan (if applicable) to allow as many buyers as possible to take advantage of your service."*

## The Construction

### 1. Start with Value Equation (Chapter 4)
Per `primitives/value-equation.md`:

```
  Value = (Dream Outcome × Perceived Likelihood) / (Time Delay × Effort & Sacrifice)
```

Score each variable 1-10. Multiply. Target ≥ 150 (Grand Slam threshold).

### 2. Specify the Dream Outcome (Chapter 5-6)
Outcomes must be:
- Specific (not "grow your business" — "add $25K MRR in 90 days")
- Identity-level (who they become, not just what they do)
- Visualizable (vivid, tangible future state)

### 3. List Likely Problems + Solutions (Chapter 7)
Brainstorm every problem the ICP faces en route to the Dream Outcome. For each, design a solution.

These solutions become VALUE STACK components or BONUSES.

### 4. Bundle into the Offer (Chapter 8)
Combine components into the core offer. Rules:
- 4-7 core components (not more — bloat dilutes perceived value)
- Each component has a named deliverable
- Standalone prices itemized

### 5. Stack Bonuses (Chapter 9)
Add bonuses strategically:
- Minimum 5 bonuses
- Each bonus counters one specific objection
- 2nd-best first (hook position)
- Best last (stack climax)
- Total bonus value 3-5× offer price

### 6. Add Scarcity + Urgency
- **Scarcity** = limited supply (cohort size, bonus limits)
- **Urgency** = limited time (price increases, bonus deadlines)

Both must be TRUE (fabricated = Truth Gate violation).

### 7. Add Guarantee (Chapter 10)
Strongest possible guarantee:
- Unconditional refund (weakest)
- Conditional outcome guarantee
- **Reversed-risk / ROI-positive** (strongest) — see `sales/value-stack-architecture.md`

### 8. Name the Offer
Specific named offer that implies the mechanism + outcome.

Examples:
- "The $25K MRR a creator-agency program"
- "Million-Dollar Coaching Method"
- "The Operator OS"

Avoid: generic names ("My Coaching Program"), jargon-only names.

### 9. Price It
Two frames:
- **Frame 1 — Value Equation math**: "Worth $X, priced at $Y (1/3 of stated value anchor)"
- **Frame 2 — ROI math**: "Returns $X within Y timeframe if buyer executes"

Price is what shows the offer is serious. Too low = not-serious signal.

## The the acquisition economist Differentiator (vs the offer architect)

Both the offer architect + the acquisition economist design high-ticket offers. Differences:

| Dimension | the offer architect | the acquisition economist |
|---|---|---|
| Primary lens | Market psychology + identity | Value equation math |
| Focus | Offer-ICP congruency | Grand Slam value stacking |
| Mechanism name | "Offer Building System" | "high-value stacked offer" |
| Pricing | Identity-aligned | Value-equation-aligned |
| Guarantee | Risk Reversal architecture | Unbeatable guarantees |
| Best for | Complex high-ticket ($10K+) | Simpler high-ticket ($2K-$50K) |

**Both work. Both are applied in /design-offer.** the offer architect provides the architecture, the acquisition economist provides the scoring.

## The Invariants (acquisition-economics methodology)

- Value > price (by widening the Value Equation)
- The simplest funnel that makes $250M+ is usually the right one (complexity kills)
- Leads are earned through the Core Four (not tricks)
- Book is the ad (education-as-marketing is the meta-mechanism)
