# Call Funnel — The Primitive

> **Source lineage:** the growth engineer (Call Funnel Blueprint), the backend economist (One-Funnel Compounding — $75M from one call funnel), the agency director (SMMA agency variant), the paid media director (Show-Rate stack on call funnels).
> **Status:** First-class primitive. The high-ticket sales substrate.

## The Canonical Path

```
Cold traffic
  ↓
VSL / Sales Page
  ↓
Application (Typeform / form)
  ↓ (qualify)
Confirmation Page + SMS + Email (the paid media director Show-Rate stack)
  ↓
Sales Call
  ↓
Close
```

Every high-ticket ($3K+) offer eventually routes through this shape — variants differ at the Application stage (some skip it = Book-a-Call variant) and at the team layer (solo creator → SDR+Closer).

## When to Use

- Offer price **$3K+**
- Creator has capacity to handle (or team to handle) live sales calls
- ICP buying process requires trust-building conversation (not impulse purchase)
- Revenue target >$30K/mo (call funnels scale with closer capacity)

## When NOT to Use

- Offer < $500 (call friction > revenue lift)
- Offer $500-$2K (self-serve checkout + VSL usually better)
- Creator has no team + no capacity for calls (build capacity first)

## The Two Variants

### Variant A — Application Funnel (higher friction, higher quality)
```
Traffic → Sales Page → Application (filters low-intent) → Book-a-Call → Call → Close
```
- Lower volume of bookings but higher close rate
- Required for $10K+ offers (qualification matters)
- Setter role optional (application + scorecard can qualify)

### Variant B — Book-a-Call Funnel (lower friction, more volume)
```
Traffic → Sales Page / VSL → Book-a-Call directly → Call → Close
```
- Higher volume + lower close rate
- Fits $3K-$10K range
- Requires setter (handle no-shows + re-books) OR automation

Per `/build-funnel`, Variant A = Archetype 3, Variant B = Archetype 4.

## The Call Funnel Blueprint

5 components (+ 1 optional):

1. **Headline** — specificity ≥ 8, outcome + timeframe
2. **VSL** (3-10 min) — teach just enough to hook, then route to application
3. **Typeform Application** — qualification + DQ criteria (5-10 questions)
4. **Qualified / DQ Confirmation Pages** — different paths based on score
5. **Setter Follow-Up** (within 5 min) — handles qualified applicants
6. **(Optional) Breakout Videos** — pre-call nurture content per question in application

## The One-Funnel Compounding Model

the backend economist's $75M+ in cash collected via **ONE** call funnel for years. Insight: rather than adding 3 more funnels, he compounded ONE to world-class:
- 10%+ conversion on VSL
- 70%+ show rate (the paid media director stack)
- 40%+ close rate on qualified shows
- $30K+ AOV
- Backend email continuity layer adding 50%+ LTV

One dialed funnel > ten mediocre. This is the `/build-funnel` INV: don't add conversion mechanism #2 until #1 is leak-free at $500K+/mo.

## The the paid media director Show-Rate Stack

See `/post-booking-nurture` skill. The 40% → 75% show-rate transformation:
- Confirmation page with creator video + homework
- SMS within 5m, 30m post, 24h, 30m pre, 5m pre
- Email 24h, 6h, 1h
- Phone triage (high-ticket only — within 5m setter double-dial)

## Per-Stage Conversion Benchmarks (with the paid media director stack)

| Stage | Benchmark |
|---|---|
| Ad → Landing Page | 2-5% CTR |
| Landing Page → Opt-in | 30-50% (warm) / 10-20% (cold) |
| Opt-in → VSL | 60-80% |
| VSL → Application Start | 10-20% |
| Application Start → Complete | 50-70% |
| Application → Qualified | 40-60% (depends on gate strictness) |
| Qualified → Booked | 70%+ (the paid media director confirmation page) |
| Booked → Showed | **75%+ (the paid media director stack) vs 40% default** |
| Showed → Closed | 25-40% |
| **End-to-End** | **0.5-2% of ad clicks** |

At $10K AOV + 1% end-to-end conversion: $100 spend = $100 revenue (break-even). $1K spend with optimization = $5K+ revenue.

## The Backend Economics Rule (backend-economics methodology)

Call funnel economics don't work on front-end alone. The **backend** (continuity, upsells, referrals, cohort expansion) is where LTV:CAC passes 3:1.

Plan:
- Front-end call funnel: break-even to 1.5:1 LTV:CAC
- Backend (continuity $297/mo + upsells + referrals): 2-4× LTV
- Combined: 3-5:1 LTV:CAC

Call funnel without backend = unprofitable scaling. Always pair.

## Validation

Call funnel is dialed when:
- [ ] End-to-end conversion 0.5%+ (good) to 2%+ (great) of ad clicks
- [ ] Show-rate 70%+ (via the paid media director stack)
- [ ] Close rate 25%+ on qualified shows
- [ ] LTV:CAC ≥ 3:1 (with backend)
- [ ] Unit economics computed (the backend economist diligence)
- [ ] SOPs exist for setter + closer + show-rate ops

## Anti-Patterns

- ❌ Skipping the paid media director show-rate stack (40% vs 75% = 88% revenue delta on identical lead volume)
- ❌ Scaling ad spend before show-rate + close-rate proven
- ❌ No backend (front-end-only = unprofitable scaling)
- ❌ Application gate too loose (closer wastes time) OR too tight (miss good buyers)
- ❌ Adding funnel #2 before #1 dialed (the backend economist rule)
- ❌ Running call funnel for <$3K offer (friction > revenue)
