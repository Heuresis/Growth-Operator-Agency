# Value Stack Architecture

> **Source lineage:** the acquisition economist (a high-value offer canon — high-value stacked offer stack) + the VSL director (15-step VSL Section 7-8-9 Features + Testimonials + Price Anchoring) + the offer architect (Engineering Economics Offer)
> **Status:** Load-bearing framework for /design-offer, /build-vsl, /webinar-script.

## The 5-Layer Close Structure

Every offer reveal has five layers stacked in sequence:

```
  Layer 1 — Core Deliverable     (what they get — the transformation)
  Layer 2 — Value Stack          (itemized components with standalone prices)
  Layer 3 — Price Anchor         (what it would cost elsewhere vs what you pay)
  Layer 4 — Guarantee            (ROI-positive risk reversal)
  Layer 5 — CTA                  (single specific action)
```

## Layer 1 — Core Deliverable

State the transformation, not the feature list:

- Bad: "12 weekly group calls + course portal + templates"
- Good: "Add $25K in monthly retainer revenue within 90 days without hiring"

Core deliverable = the ONE thing the offer does. Frame at identity level (per the offer architect — "they buy the identity that uses this").

## Layer 2 — Value Stack (itemized)

Break the offer into components with STANDALONE prices. Each component:
- Named (short, sticky)
- Described in 1-2 lines
- Priced separately ("If you hired this standalone: $X")
- Sums to 3-5× the offer price (anchor)

```
| Component | What it is | Standalone value |
|---|---|---|
| [Named component 1] | [1-2 lines] | $5,000 |
| [Named component 2] | [1-2 lines] | $3,500 |
| [Named component 3] | [1-2 lines] | $4,200 |
| [Named component 4] | [1-2 lines] | $2,800 |
| [Named component 5] | [1-2 lines] | $1,900 |
| **Total stated value** | | **$17,400** |
```

Rule: **3-5× the offer price.** More = not believable. Less = weak anchor.

## Layer 3 — Price Anchor

Anchor against SPECIFIC alternatives, not vague comparisons:

- DIY cost: "If you try to figure this out alone, it'll take 6-12 months + $15K in trial-and-error tools"
- Done-for-you competitor: "Agencies charge $8K-$15K/month to build this for you"
- Status-quo cost: "Every month without this = $X in lost revenue based on ICP math"

Then reveal the offer price: "Total stated value $17,400. Investment today: $4,997."

Arithmetic, not vague. Specific numbers or reject.

## Layer 4 — Guarantee (ROI-Positive)

Structure the guarantee to be ROI-positive — buyer makes more than offer price even if it "fails":

### Tier 1 — Unconditional refund (baseline, weak)
30-day money-back. Most creators stop here. Doesn't move needle.

### Tier 2 — Conditional outcome guarantee (stronger)
"Get X result in Y timeframe or refund"

### Tier 3 — Reversed-risk guarantee (strong)
"Get X result in Y timeframe or I work with you until you do."

### Tier 4 — Better-than-zero guarantee (strongest)
"Get X in Y or refund + you keep [valuable bonus]."

### Tier 5 — Pay-for-results guarantee (nuclear)
"You only pay the full price after you hit [milestone]."

**Rule:** guarantee tier matches offer tier. $997 can get away with Tier 1-2. $10K+ should hit Tier 3-5.

## Layer 5 — CTA (single specific action)

ONE action. Not three. Specific verb. Direct link / button / CTA.

- Bad: "Visit our site, book a call, or email us"
- Good: "Book your call here: [link]"

Action verbs by offer type:
- Call funnel → "Book your call"
- VSL checkout → "Get access"
- Application funnel → "Apply now"
- Challenge → "Join the challenge"
- Tripwire → "Grab it"

## The Bonus Stacking Sub-Architecture

Bonuses sit between Layer 2 (Value Stack) and Layer 3 (Price Anchor). Rules per `/design-offer`:

- **5+ bonuses** minimum (acquisition-economics methodology)
- **2nd-best bonus FIRST** (hook position)
- **Best bonus LAST** (stack climax)
- **Middle bonuses = objection counters** (each counters one ICP Section 10 objection)
- **Each bonus has standalone stated value** (adds to value stack total)

## The Reveal Sequence (timing matters)

In a VSL or webinar pitch, reveal in this exact order:

1. Core deliverable (5-10% of pitch runtime)
2. Value Stack components with standalone prices (30-40%)
3. Price anchor reveal (10%)
4. Bonus stack (best-last) (20-25%)
5. Guarantee reveal (10%)
6. Urgency + Scarcity (if applicable) (5%)
7. Single CTA (5%)

**Rule:** never reveal price before showing value. Price before value = sticker shock.

## Tacit Principles

1. **Stated value ≠ real value** — this is the anchor game. Real-value delivery is a different question (that's the product).

2. **Itemization anchors.** Bundling without itemization = less perceived value.

3. **Guarantee creates the sale; product delivers it.** Strong guarantee = high conversion. Weak guarantee = attendees "think about it."

4. **Pricing is identity.** $5K and $50K offers require different identity markers even if delivering similar outcomes. See the offer architect pricing-identity rule.

5. **Bonuses kill objections.** Every bonus maps to one objection. If you can't map, it's bloat.

6. **Read the Value Stack aloud.** If each component doesn't sound distinct and worth its stated price, revise.

7. **Don't stack past 7 components.** 3-5 is the sweet spot. 7+ dilutes.
