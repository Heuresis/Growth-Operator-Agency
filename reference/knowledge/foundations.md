# Foundations — Shared Knowledge

## Purpose
Shared vocabulary, frameworks, KPIs, and handoff patterns inherited by every skill in the Foundations department. Before anything else, we make the ideal customer, the positioning, the voice, and the offer legible on paper.

## Department Skills (5)
- `/research` — deep-dive discovery into market, niche, customer language
- `/build-icp` — codify Ideal Customer Profile (demographics + psychographics + behaviors)
- `/build-positioning` — lock differentiation + category + angle
- `/extract-voice` — capture founder voice patterns, lexicon, cadence
- `/design-offer` — architect the flagship offer via Value Equation

## Core Vocabulary
- **ICP** — Ideal Customer Profile. One named, one-page avatar with triggers, pain tiers (1–5), and buying criteria.
- **LTV:CAC** — Lifetime value to customer acquisition cost. Target ≥ 3:1 for healthy unit economics.
- **Value Equation** (acquisition-economics methodology) — `(Dream Outcome × Perceived Likelihood) / (Time Delay × Effort & Sacrifice)`. Score each lever 1–10, multiply numerator, divide denominator. Target ≥ 150.
- **8 Required Beliefs** — Beliefs a prospect must hold to buy: (1) problem is real, (2) problem is urgent, (3) status quo is failing, (4) external solution exists, (5) your category is the right one, (6) your mechanism is superior, (7) you are credible, (8) now is the time.
- **Limiting Belief Triad** — (a) "It won't work for me", (b) "I can't do it", (c) "It's not worth the risk". Every piece of content must dissolve one.
- **Signal-to-Noise (S/N)** — Ratio of on-brief insight to filler in research output. Target ≥ 0.8.
- **Completeness Score** — % of ICP fields with cited evidence. Target ≥ 90%.
- **Offer Anatomy** — Core Promise + Mechanism + Proof + Stack + Guarantee + Price + Scarcity.

## Core KPIs
| KPI | Target | Measured By |
|---|---|---|
| Research Completeness | ≥ 90% fields cited | `/research` |
| Signal-to-Noise | ≥ 0.8 | Reviewer pass |
| ICP Named + One-Pager | 100% complete | `/build-icp` |
| Value Equation Score | ≥ 150 | `/design-offer` |
| Positioning Differentiation Test | Passes "only-we" + "category" + "enemy" | `/build-positioning` |
| Voice Fidelity | ≥ 8/10 blind-author test | `/extract-voice` |

## Handoff Patterns
1. `/research` → produces `research-brief.md` with cited customer-language pulls. **Gate:** S/N ≥ 0.8 before `/build-icp` runs.
2. `/build-icp` → produces `icp.md` (one avatar, one page). **Gate:** Completeness ≥ 90% before `/build-positioning`.
3. `/build-positioning` → produces `positioning.md` (category, enemy, only-we statement). **Gate:** 3 differentiation tests passed before `/extract-voice`.
4. `/extract-voice` → produces `voice-guide.md` (20+ phrases, 10+ words, cadence rules). **Gate:** 8/10 blind-author test before `/design-offer`.
5. `/design-offer` → produces `offer.md` with Value Equation ≥ 150. **Gate:** Offer must name the Limiting Belief it dissolves + which of 8 Required Beliefs it anchors.

## Patterns We Use
- **Customer-language capture**: pull 50+ verbatim quotes per research sprint (forums, reviews, transcripts, DMs). Tag each with pain tier.
- **Pain tiers (1–5)**: 1 = nuisance, 3 = recurring cost, 5 = identity-threatening. Offer must target ≥ 4.
- **One-page ICP rule**: if it doesn't fit one page, it isn't tight enough.
- **"Only-we" statement**: `We are the only [category] that [mechanism] for [ICP] who want [dream outcome] without [effort/sacrifice they fear].`
- **Voice cadence**: sentence-length distribution, rhetorical patterns (rule of 3, contrast pairs), signature words.
- **Offer stack math**: every bonus must add ≥ $500 perceived value and dissolve ≥ 1 belief gap.

## Anti-patterns (block these)
- Demographic-only ICPs (no psychographics = no leverage)
- Positioning via "better/faster/cheaper" (not differentiation)
- Voice guides with adjectives only ("friendly, approachable") — need phrases + cadence
- Offers with < 3 proof elements
- Value Equation scored without documented reasoning
