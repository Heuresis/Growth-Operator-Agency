# `reference/` — Methodology library

The structured-knowledge layer (Layer 2 of the architecture). Every `SKILL.md` body cites into here rather than carrying its methodology inline.

| Subdirectory | Holds |
|---|---|
| [`canonical/`](canonical/) | Canonical-frameworks index — classical direct-response, awareness-spectrum, market-sophistication, market-hierarchy, influence principles, cult methodology |
| [`frameworks/`](frameworks/) | Active framework files used by skills — acquisition economics, esoteric marketing, esoteric offer architecture, VSL, YouTube, Instagram funnel, sales, primitives |
| [`operators/`](operators/) | Anonymized operator playbooks — methodology descriptors only (the acquisition economist, the offer architect, the agency director, the VSL copywriter, etc.) |
| [`operator-research/`](operator-research/) | Staging area for raw research notes before encoding |
| [`competitors/`](competitors/) | Competitor teardown notes |
| [`data-sources/`](data-sources/) | Where to pull market signal from (forums, communities, search APIs) |
| [`examples/`](examples/) | Worked-example deliverables (positioning docs, ICP docs, offer docs, signal reports) |
| [`integrations/`](integrations/) | External-tool bindings (CRM, email, ad platforms, scheduling) |
| [`knowledge/`](knowledge/) | Cross-cutting domain knowledge per department |
| [`legal-templates/`](legal-templates/) | Contracts and agreements — affiliate, JV, retainer, etc. |
| [`platforms/`](platforms/) | Platform-specific encoding — YouTube, Instagram, X, LinkedIn |
| [`playbooks/`](playbooks/) | Multi-skill orchestration recipes (e.g. mentorship-funnel) |
| [`swipe-file/`](swipe-file/) | Anatomized examples — hooks, posts, threads, stories, testimonials, case studies |
| [`templates/`](templates/) | Fill-in-the-blank artifact templates |
| [`verticals/`](verticals/) | Per-vertical playbooks — coaching, agency, info-course, e-com, fitness, real-estate, etc. |
