# Insight Hooks

> **25 insight/reframe hooks.** Non-obvious observations. Save-optimized. Earn "I hadn't thought about it that way."

---

## How Insight Hooks Work

An insight = an observation the reader's future self will quote. The test: can the reader imagine repeating this to a friend? If yes, it's an insight. If it's a restatement of common knowledge, it isn't.

Three insight patterns:
1. **The Hidden Pattern** — connect two things the reader hadn't connected
2. **The Surprising Principle** — state a law the reader hasn't formalized
3. **The Reframe That Changes Action** — shift the reader's operating frame

---

## Variant Structure Template

```
[COMPRESSION — one sentence containing the full insight]
[PROOF OR EXAMPLE — concrete evidence in 1-2 lines]
[OPTIONAL: implication for the reader's behavior]
```

Compress. Insight hooks work because they say more in fewer words. If you take three lines to say the insight, it isn't compressed enough yet.

## The Hooks

### Pattern: The Hidden Pattern

**1. The X = Y Formulation**
```
[THING A] is just [THING B] with [DIFFERENTIATOR].
Once you see it, you can't unsee it, and [IMPLICATION].
```
Platform fit: LinkedIn, X, YouTube title.

**2. Two Things Are The Same Thing**
```
[DIFFERENT-LOOKING ACTIVITY A] and [DIFFERENT-LOOKING ACTIVITY B] are the same activity.
Both are [UNDERLYING THING]. Which means you already know how to do both.
```
Platform fit: LinkedIn, YouTube, X.

**3. Category Confusion**
```
What [ICP] call [CATEGORY NAME] is actually [DIFFERENT CATEGORY].
That reassignment changes who competes with you and how you market.
```
Platform fit: LinkedIn, YouTube, X.

**4. The Same Shape**
```
[PHENOMENON A], [PHENOMENON B], [PHENOMENON C] — all have the same shape.
The shape is [UNDERLYING STRUCTURE]. One fix works on all three.
```
Platform fit: LinkedIn, YouTube, carousel.

**5. Pattern Name**
```
This pattern doesn't have a name, so I'll give it one: [NEW NAME].
It's: [ONE-LINE DESCRIPTION]. Once named, it's everywhere.
```
Platform fit: LinkedIn, YouTube, X.

**6. Shape Recognition**
```
You're looking at [SURFACE PROBLEM]. The shape is [UNDERLYING PROBLEM].
Mistaking the shape is why the fix keeps missing.
```
Platform fit: LinkedIn, YouTube, X.

**7. Hidden Dependency**
```
[THING A] only works because of [UNNAMED DEPENDENCY B].
Remove B, A collapses. Most [ICP] optimize A and ignore B.
```
Platform fit: LinkedIn, YouTube, X.

### Pattern: The Surprising Principle

**8. The Law Nobody States**
```
[UNSTATED LAW OF THE CATEGORY]: [CONCRETE PRINCIPLE].
Everyone operates by it. Nobody teaches it.
```
Platform fit: LinkedIn, YouTube, X.

**9. The Tradeoff Nobody Names**
```
[THING ICP WANT A] and [THING ICP WANT B] can't both max out.
Picking which to max is a design decision most [ICP] avoid.
```
Platform fit: LinkedIn, YouTube, X.

**10. The Minimum Viable Thing**
```
The minimum [ICP] need to start producing [OUTCOME] is [LOWER-THAN-EXPECTED INPUT].
They've been told they need [INFLATED INPUT]. They don't.
```
Platform fit: LinkedIn, YouTube, X.

**11. The Asymmetric Payoff**
```
[SMALL ACTION X] has a [LARGE OUTCOME Y] payoff structure.
[LARGE ACTION Z] has a [SMALL OUTCOME W] payoff structure. Most [ICP] pick wrong.
```
Platform fit: LinkedIn, X, YouTube.

**12. The Compounding Variable**
```
[VARIABLE X] compounds. [VARIABLE Y] doesn't. They look equal at [EARLY STAGE].
At [LATER STAGE], the gap is [N×].
```
Platform fit: LinkedIn, YouTube, X.

**13. The Diminishing Return**
```
After [THRESHOLD], each additional unit of [INPUT] produces less [OUTPUT].
Most [ICP] push past the threshold. The right move is to cap input and change the system.
```
Platform fit: LinkedIn, YouTube, X.

**14. Second-Order Effect**
```
Doing [OBVIOUS ACTION X] produces [EXPECTED OUTCOME Y].
It also produces [UNEXPECTED SECOND-ORDER EFFECT Z]. Z is usually bigger than Y.
```
Platform fit: LinkedIn, YouTube, X.

**15. The Selection Effect**
```
[OUTCOME] isn't produced by [ASSUMED CAUSE].
It's produced by [SELECTION — who enters the room]. Change the selection, change the outcome.
```
Platform fit: LinkedIn, YouTube, X.

### Pattern: The Reframe That Changes Action

**16. Reframe Verb**
```
Stop calling it "[COMMON VERB]." Call it "[BETTER VERB]."
That word change shifts the entire operation.
```
Platform fit: LinkedIn, X, YouTube.

**17. The Constraint Reframe**
```
[THING ICP TREAT AS PROBLEM] isn't a problem. It's the constraint that makes everything work.
Remove it, quality collapses. Protect it, compound.
```
Platform fit: LinkedIn, YouTube, X.

**18. Problem-To-Asset Reframe**
```
Your [PERCEIVED WEAKNESS] is actually the exact asset [RIGHT AUDIENCE] is hiring for.
Stop hiding it. Lead with it.
```
Platform fit: LinkedIn, YouTube, X.

**19. Scarcity-To-Leverage Reframe**
```
You have [LESS OF COMMON RESOURCE] than competitors.
That's leverage, not deficit. Here's why and how.
```
Platform fit: LinkedIn, YouTube, X.

**20. The Inverted Question**
```
Instead of asking "how do I [COMMON QUESTION]?", ask "[INVERTED QUESTION]?"
Same problem. Different answer. Different life.
```
Platform fit: LinkedIn, YouTube, X.

**21. The Smaller Goal Reframe**
```
[LARGER GOAL] is the wrong target.
[SMALLER, SPECIFIC GOAL] is. Hit the smaller one and the larger one follows. Chase the larger one directly and you miss both.
```
Platform fit: LinkedIn, YouTube, X.

**22. The Systems-Not-Goals Reframe**
```
Set a system, not a goal.
[SPECIFIC SYSTEM DESCRIPTION]. The goal becomes a byproduct.
```
Platform fit: LinkedIn, YouTube, X.

**23. The Who-Not-How Reframe**
```
You don't have a [HOW] problem. You have a [WHO] problem.
The [HOW] gets solved when the right [WHO] enters the room.
```
Platform fit: LinkedIn, YouTube, X.

**24. Near-Term / Long-Term Reframe**
```
Optimizing for [NEAR-TERM METRIC] is optimizing against [LONG-TERM METRIC].
Pick the horizon, then pick the metric. Not the other way around.
```
Platform fit: LinkedIn, YouTube, X.

**25. Identity-Before-Action Reframe**
```
You're trying to [ACTION YOUR WAY TO OUTCOME].
Identity has to shift first. [IDENTITY] people do [ACTION] without friction. Force the action without the identity = burnout.
```
Platform fit: LinkedIn, YouTube, email SL.

## Insight Density Test

A true insight passes this test: cover the reader's proof/example and can they still guess what the insight implies? If they can, the insight is load-bearing. If they can't, the insight was propped up by examples and isn't durable.

---

## The Insight-To-Save Pipeline

Insight hooks drive saves because:
- Reader doesn't want to lose the sentence
- Reader knows they'll want to quote it later
- The sentence is compressed enough to re-read in 3 seconds

Structural moves that raise save rate:
- Bold the compressed sentence
- Leave the one-line insight on its own line
- Include a "save this for when [SPECIFIC MOMENT]" close

---

## Combination Rule

Insight hooks pair with:
- Contrarian hooks (insight = the reframe behind the reversal)
- Specific-result hooks (insight + proof)
- Story hooks (narrative → earned insight)

Never pair with another insight hook. One insight per asset. Two insights dilute each other.
