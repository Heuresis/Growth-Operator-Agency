# Specific Result Hooks

> **25 specific-result proof hooks.** Number first. Story behind. Specificity = permission to make the claim.

---

## How Specific-Result Hooks Work

The reader scans for proof signals in the first line. Specific numbers pass the smell test; round numbers don't. "$47,200 in 11 days" earns attention; "$50k in ~2 weeks" doesn't.

Three proof patterns:
1. **Dollar Forward** — lead with the revenue figure
2. **Outcome Forward** — lead with the non-monetary result
3. **Constraint Forward** — lead with what was missing or limiting

---

## Variant Structure Template

```
[SPECIFIC NUMBER + SPECIFIC TIMEFRAME + SPECIFIC ACTIVITY]
[REVEAL ANGLE — cost, effort, tool, origin point, or constraint]
[OPTIONAL: authority marker or replicability promise]
```

Round numbers are the enemy. $10,000 → $9,847. 30 days → 27 days. 100 clients → 94 clients. Specificity wins.

## The Hooks

### Pattern: Dollar Forward

**1. Revenue + Timeframe + Method**
```
$[SPECIFIC AMOUNT] in [SPECIFIC TIMEFRAME] using [SPECIFIC METHOD].
Here's the breakdown — including what didn't work.
```
Platform fit: Reels, LinkedIn, X, YouTube short.

**2. Revenue Without**
```
$[SPECIFIC AMOUNT] collected [TIMEFRAME] without [COMMON REQUIREMENT].
The [REQUIREMENT] was the thing I thought I needed. I didn't.
```
Platform fit: LinkedIn, Reels, X.

**3. Revenue Per Hour**
```
$[SPECIFIC AMOUNT] for [SPECIFIC HOURS] of work.
That's $[COMPUTED HOURLY] per hour. The system behind it is simpler than expected.
```
Platform fit: LinkedIn, Reels, X.

**4. Revenue From One Asset**
```
One [ASSET TYPE] generated $[SPECIFIC AMOUNT] over [TIMEFRAME].
Total time to build it: [HOURS]. Below: what's inside.
```
Platform fit: LinkedIn, Reels, YouTube.

**5. Revenue Shift**
```
Went from $[BEFORE] to $[AFTER] in [TIMEFRAME] by removing [COMMON ACTIVITY] and adding [SPECIFIC ACTIVITY].
```
Platform fit: LinkedIn, YouTube long-form title, X.

**6. Revenue With Counter-Intuitive Input**
```
$[SPECIFIC AMOUNT] generated in [TIMEFRAME] by working [COUNTER-INTUITIVE METRIC — fewer hours, smaller audience, etc.].
The constraint forced the leverage.
```
Platform fit: LinkedIn, Reels, YouTube.

**7. ROI Receipts**
```
Spent $[SPECIFIC AMOUNT] on [ACTIVITY]. Returned $[SPECIFIC RETURN] in [TIMEFRAME].
ROI: [RATIO]. Here's what the $[SPENT] actually bought.
```
Platform fit: LinkedIn, X, YouTube.

**8. The Surprising Margin**
```
[%] margin on $[REVENUE] over [TIMEFRAME].
Most [ICP TYPE] assume margins cap at [LOWER %]. Here's the structural why.
```
Platform fit: LinkedIn, YouTube, X.

## Proof Integrity Rule

Every specific-result hook must be backed by:
1. **Receipt** — screenshot, invoice, dashboard, dated artifact
2. **Date** — when the result was produced
3. **Method** — the actual replicable steps
4. **Constraint** — what was true about the situation (so the reader can match)

Violations (from `INVARIANTS.md`):
- Never invent numbers
- Never round in a way that inflates
- Never claim results without timeframe
- Never use cumulative lifetime numbers without saying so

## Specificity Math

A hook's Specificity Score is computed as:

| Element | Points |
|---|---|
| Specific number (not rounded) | +3 |
| Specific timeframe | +2 |
| Named activity / method / mechanism | +2 |
| Identity marker (ICP type) | +1 |
| Authority marker or delta | +1 |
| Base | +1 |

Max 10. Target for publishing: ≥ 8.

Example:
- "I made $47,200 in 11 days with a one-page PDF as a new consultant after dropping my retainer model" = 3+2+2+1+1+1 = 10 ✓
- "I made some money with a PDF" = 1 ✗

---

## Combination Rule

Specific-result hooks pair with:
- Curiosity hooks in line 2 (proof → gap about how)
- Story hooks in the body (proof → backstory)
- Insight hooks in the body (proof → hidden mechanism)

Never pair with another specific-result hook. Stacking numbers in the hook slot = reader can't anchor.
