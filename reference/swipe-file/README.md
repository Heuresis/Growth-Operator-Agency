# swipe-file — Remixable Content Library

> **Purpose:** battle-tested skeletons, templates, and openers the creator can remix into voice-matched assets. Not scripts. Not one-size-fits-all. Starting clay the skills shape.
>
> **Scope:** hooks, LinkedIn post structures, X thread skeletons, IG story sequences, case-study structures, testimonial formats, email subject-line systems.
>
> **Rule:** every swipe file item is a **structural template** with slots, not a ready-to-paste line. Fill with the creator's ICP, offer, voice.

## How to Use

### Step 1 — Pick the asset type
You're writing a LinkedIn post. Go to `posts-linkedin/`. You're writing the opener for a Reel. Go to `hooks/`. You're structuring a case study. Go to `case-studies/`.

### Step 2 — Pick the closest structural match
Read the `_INDEX.md` in each folder. Each index classifies the templates by intent (curiosity vs proof vs contrarian vs identity) and by platform fit.

### Step 3 — Fill the slots
Every template has `[BRACKETED SLOTS]`. Replace each with ICP-specific, offer-specific, voice-specific content. Never paste a slot unfilled. Never paste a slot with generic filler.

### Step 4 — Voice-match
Run the filled draft through:
- `phrases_to_use` from Compartment 1 (include 3+)
- `phrases_to_avoid` sweep
- `spec/BANNED-VOCABULARY.md` regex scan

### Step 5 — Specificity check
Hooks: specificity ≥ 8. Case studies: named outcome + timeframe + mechanism. Testimonials: dated + dollar figure if monetary + identity shift marker if non-monetary.

### Step 6 — Ship through the right skill
Swipe file is **input**, not output. Feed it to `/write-linkedin-post`, `/write-x-thread`, `/write-reel`, `/case-study`, `/email-sequence` for voice-match + verification + evidence gate compliance.

## Index Pattern

Every folder follows the same index pattern:

```
category-name.md          — main file with templates
_INDEX.md                 — lists all categories + counts + when to pick each
```

---

## Constraint Notes

Every template in this swipe file:
- No real operator names (uses archetypes: "the acquisition economist", "the short-form-frameworks author", etc.)
- No real brand names (uses ICP-category instead: "fitness coach", "B2B consultant")
- No copyrighted framework names (uses functional descriptors: "Hook-Loop-CTA" instead of branded names, "3-A story" instead of branded)
- No @handles, URLs, client names, or Notion UUIDs
- Passes `spec/BANNED-VOCABULARY.md` regex scan

---

## Refresh Cycle

Swipe-file items are **durable structure** (Layer 2), not ephemeral copy (Layer 3). Structural templates survive platform changes. When a platform update invalidates an entry (e.g., LinkedIn Dec 2025 Knowledge Graph shift), update the structural template here. Copy-level refresh happens in the skill's adapter files.
