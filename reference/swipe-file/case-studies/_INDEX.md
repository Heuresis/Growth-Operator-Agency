# case-studies — Swipe File Index

> **10 case-study structures.** Each is a reusable scaffold for telling a client result. Matches `/case-study` skill.

## Structure Map

| File | Structure | Best When |
|---|---|---|
| `before-after.md` | Classic delta — starting state → ending state | Numbers-rich cases, clear metric |
| `obstacle-decision-outcome.md` | ODO — obstacle faced → decision made → outcome produced | Decision-focused lesson |
| `hero-cycle.md` | Call → refusal → mentor → transformation → return | Narrative-heavy cases |
| `isomorphic.md` | Same shape + similar situation + same outcome (from `INVARIANTS.md`) | Replicability pitch |
| `reverse-order.md` | Start with outcome, walk backward to starting state | Hook-first telling |
| `numbers-first.md` | Lead with 3-5 numbers, then story | Proof-dominant case |
| `skeptic-convert.md` | Client was skeptical → process → convinced | Objection-handling case |
| `compound-result.md` | Small wins → compound over time → large outcome | Long-timeframe cases |
| `team-led.md` | Client's team (not client personally) executed → outcome | Team-sold offers (agencies, B2B) |
| `quiet-operator.md` | Client who doesn't want attention → quiet results → transferable principle | Private-client cases |

**Total: 10 structures. Each in its own file.**

## When To Pick Which

### Pick Before-After when
- You have clean delta math
- Numbers are publishable
- Audience is proof-seeking (decision stage)

### Pick Obstacle-Decision-Outcome when
- The lesson is about the decision, not the result
- The decision is transferable beyond this client
- You want to teach judgment, not tactics

### Pick Hero Cycle when
- The story has real narrative arc
- The client's identity transformed (not just metrics)
- Long-form format (LinkedIn, YouTube, email)

### Pick Isomorphic when
- You're trying to show "your story could be their story"
- Audience needs to see themselves in the client
- Best for pre-launch warming

### Pick Reverse Order when
- You want hook-first punch
- Delta is dramatic enough to open with
- Platform: Reels, X thread, IG story

### Pick Numbers First when
- Proof is the whole point
- Audience is sophisticated / data-forward
- Short-form platform: X, Reels

### Pick Skeptic Convert when
- The client had objections similar to your current prospects
- Objection-handling is the asset
- Audience is at the objection-prone stage

### Pick Compound Result when
- Timeframe > 12 months
- Win pattern is small-wins-compounding
- Audience: impatient / looking-for-shortcut types (case teaches patience)

### Pick Team Led when
- Client delegated execution
- Offer serves organizations, not individuals
- B2B / enterprise ICP

### Pick Quiet Operator when
- Client doesn't want public attention
- You can extract principles without identifying
- Audience: private / non-performer types

## Case Study Integrity

Per `INVARIANTS.md`:
- Never invent cases
- Never manufacture numbers
- Always get permission before identifying
- Always preserve client anonymity when preferred
- Always include timeframe
- Always include what-made-it-work mechanism

## Distribution Surface

A single case study can feed:
- 1 LinkedIn post (full form)
- 1 X thread (condensed version)
- 1 Reel / Short (60-sec cut)
- 1 Email broadcast (narrative form)
- 1 IG story sequence (7-day per `case-study-story.md`)
- 1 Sales page case study block
