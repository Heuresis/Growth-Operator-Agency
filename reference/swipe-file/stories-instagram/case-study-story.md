# 7-Day Case Study Story Sequence — Instagram

> **One full client result walkthrough across 7 days.** Proof-forward. Bridges to offer inquiry.

---

## When To Use

- Fresh client result ready (with permission)
- Proof-forward week
- Conversion-pillar slot
- Bridge to a consultative offer

---

## The 7-Day Arc

| Day | Theme | What You Show |
|---|---|---|
| Monday | Before state | Where the client started |
| Tuesday | The first diagnostic | What was actually wrong |
| Wednesday | The first move | What we did in week 1 |
| Thursday | The turn | The specific pivot moment |
| Friday | The result | Dated numbers, outcome |
| Saturday | The mechanism | Why it worked — transferable |
| Sunday | The offer bridge | Soft CTA tied to the mechanism |

---

## Monday — Before State

**Hook:**
```
Frame 1: "Showing you one [ICP]'s 90-day arc over the next 7 days."
Frame 2: "Today: where they started. Unedited."
```

**Value:**
```
Frame 3: "Starting state: [ANONYMIZED ICP DESCRIPTION]"
Frame 4: "Revenue then: $[AMOUNT]/mo"
Frame 5: "Time worked then: [HOURS]/week"
Frame 6: "Core frustration they named on intake: '[PARAPHRASED QUOTE]'"
Frame 7: "Why I took them on: [SPECIFIC FIT REASON]"
```

**Connection:**
```
Slider: "How close is this to your starting state?" — Far / Similar / That's me
```

**CTA:**
```
"Tomorrow: what was actually broken. Notifications on."
```

## Tuesday — The First Diagnostic

**Hook:**
```

```

**Value:**


**CTA:**
```
"Tomorrow: the first move we made. Week 1 of the work."
```

## Saturday — The Mechanism

**Hook:**
```
Frame 1: "The mechanism — why this specific arc worked."

```

**Value:**
```
Frame 3: "Principle: [ONE SENTENCE COMPRESSION]"
Frame 4: "Why it works: [1-2 sentences]"
Frame 5: "Where it breaks: [HONEST CAVEAT]"
Frame 6: "Who it works for: [SPECIFIC ICP CRITERIA]"

```

**Connection:**
```

```

**CTA:**
```
"Tomorrow: if this is your story, here's the move."
```

## Sunday — The Offer Bridge

**Hook:**
```
Frame 1: "If this case study felt like reading your own story — today's for you."
Frame 2: "Straightforward note, no hype."
```

**Value:**
```
Frame 3: "I work with ~[N] [ICP] at a time on engagements like the one I just walked through."
Frame 4: "The engagement is [DURATION], starts at [PRICE RANGE — honest if it's fair to share]"
Frame 5: "The criteria for fit: [3-4 BULLETS]"
Frame 6: "The process to start: [SPECIFIC STEP — e.g., 'apply via link' or 'DM with 1 sentence about your situation']"
Frame 7: "No pressure, no hype. If it's right, it's right. If not, the case study itself was the value."
```

**Connection:**
```
Question box: "Any question before you DM or skip?"
```

**CTA:**
```
Frame 8: "If you want to talk, DM 'apply' and I'll send the short form."
[Link sticker optional — to application page]
```

## Slot-Fill Rules

- Client permission REQUIRED before starting the sequence
- Anonymize if needed — use archetype description instead of name
- Numbers must be real (never fabricate)
- Quote must be real (paraphrase OK if client approved paraphrase)
- Bridge to offer must be soft on Sunday — this is proof content, not pitch content

---

## Proof Integrity

Per `INVARIANTS.md`:
- Never invent case studies
- Never manufacture quotes
- Never round numbers in inflating ways
- Always get written permission from the client before sequence starts
- Always mention if identifying details are altered

---

## Visual Rhythm

- Monday: mixed lo-fi + text (introducing the person)
- Tuesday-Friday: data-forward, specific numbers prominent
- Saturday: insight/teaching frame style
- Sunday: quieter, direct, no high-design

---

## Common Failure Modes

- Running sequence without explicit client permission
- Hype language ("life-changing results") — use specificity instead
- Sunday bridge too hard / pitchy
- No anti-ICP specified (sequence feels universal → less credible)
- Missing mechanism day (Saturday skipped)
