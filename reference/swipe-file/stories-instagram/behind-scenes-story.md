# 7-Day Behind-The-Scenes Story Sequence — Instagram

> **Trust-building. Transparency. Humanization.** 7-day sequence that shows the real shape of the creator's operation.

---

## When To Use

- Warm audience that already knows the offer
- Pre-launch warming (3-6 weeks out)
- Building trust before a higher-ticket consultative offer
- Newer account establishing authenticity

---

## The 7-Day Arc

| Day | Theme | What You Show |
|---|---|---|
| Monday | Week ahead | How you plan the week |
| Tuesday | Deep work session | What real work looks like |
| Wednesday | Tools + stack | Your actual operational stack |
| Thursday | Meetings + calls | A sanitized look at how you operate |
| Friday | A failure / rough moment | Something real that went wrong |
| Saturday | The decision process | How you decide between options |
| Sunday | Week in numbers | Honest metric review |

---

## Monday — Week Ahead

**Hook:**
```
Frame 1: "Monday. Here's what the week actually looks like."
Frame 2: "Not the Instagram version. The real version."
```

**Value:**
```
Frame 3: "[Photo of actual notebook / planning tool]"
Frame 4: "One word for the week: [SPECIFIC WORD]"
Frame 5: "Top 3 moves this week: [LIST]"
Frame 6: "What I'm protecting: [SPECIFIC BLOCK — e.g., Thursday meeting-free day]"
Frame 7: "What I'm cutting: [SPECIFIC DEPRIORITIZATION]"
```

**Connection:**
```
Poll: "Are you also running a meeting-free day?" — Yes / Never tried / Failing at it
```

**CTA:**
```
"I'll post the actual week against the plan on Sunday."
```

## Tuesday — Deep Work Session

**Hook:**
```
Frame 1: "90 minutes of actual deep work starts in 3 minutes."
Frame 2: "Documenting it because I want you to see what it actually looks like."
```

**Value (5-7 frames spread across the day):**
```
Frame 3: "Pre-block: desk clear, one doc open, timer set, phone in another room"
Frame 4 [during]: "[Screenshot of work in progress — sanitized]"
Frame 5 [mid-block]: "Deep work isn't flow. It's friction. I almost stopped twice."
Frame 6 [post-block]: "Output: [SPECIFIC DELIVERABLE]"
Frame 7: "What it felt like: [HONEST ANSWER — e.g., hard, grinding, satisfying only after]"
```

**Connection:**
```
Question box: "When's your last 90-minute block?"
```

**CTA:**
```
"Save this for your next attempt."
```

## Slot-Fill Rules

- Real numbers only (no manufacturing)
- Sanitize sensitive info (client names, specific deals, private team matters)
- Honest tone — if something was hard, say so
- Consistency in posting time (readers come to expect the rhythm)

---

## Boundary Rules

BTS stories walk a line between transparency and oversharing. Boundaries:
- Never share client information without explicit permission
- Never share team member internals (conflicts, performance issues) publicly
- Never share financial info about other people
- Never share specific revenue numbers you'd regret tomorrow

When in doubt, sanitize or skip.

---

## Visual Rhythm

Lo-fi style across the week:
- Photos of real workspace (not staged)
- Screenshots of actual notes (sanitized)
- Voice-to-camera fragments
- Consistent but loose palette (not tightly designed like launch sequence)

---

## Common Failure Modes

- Too polished (reads as performance, not transparency)
- Oversharing (breaks client / team trust)
- Fake humility ("my worst day was making $47k")
- Pitching in the middle of BTS (breaks the frame)
- Inconsistent cadence (readers tune out if Wednesday never happens)
