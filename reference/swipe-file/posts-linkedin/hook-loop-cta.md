# Hook-Loop-CTA Structure — LinkedIn

> **Hook → Loop (value body) → CTA.** Default LinkedIn structure. Works across all pillars. Easiest to adapt from hook swipe file.

---

## The Structure

```
[HOOK]
First 1-2 lines. ≤140 chars before "see more." 
Specificity ≥ 8. Opens a loop the reader must see closed.

[LOOP / BODY]
Keeps tension while delivering value. 
Format choices: paragraph, bullet list, numbered list, short scenes, Q&A.
Target: 1200-1800 chars body.

[CTA]
One action. Reply / comment / DM / follow / link-in-comments.
No stacking.
```

### When To Use
- Default choice when PIER / Listicle / Personal-to-Business don't fit
- Offer-tied conversion posts (Friday cadence)
- Lead Magnet posts (comment X, I DM you Y)
- Any pillar

## Worked Example 1 — Value Bomb (Authority)

```
[HOOK]
Spent [$X] on [CATEGORY] in the last 90 days.
Tracked every return. Here's what produced [N%] of the revenue.

[LOOP]
There were 6 activities. Not all were equal.

Activity A (40% of spend): returned [X]% of revenue. Kept.
Activity B (20% of spend): returned [X]% of revenue. Doubled.
Activity C (15% of spend): returned [X]% of revenue. Killed.
Activity D (10% of spend): returned [X]% of revenue. Killed.
Activity E (10% of spend): returned [X]% of revenue. Experimenting.
Activity F (5% of spend): returned [X]% of revenue. Killed.

Three observations from the data:

1. The highest-return activity (B) was the one I almost killed in Q2 because it "felt slow."
2. The activities that felt most productive (D, F) returned the least.
3. Concentration beat diversification. Doubling B moved the needle more than running all six.

Save this when you do your next quarterly review — audit every activity against the same two columns.

[CTA]
Want the tracking template I used? Comment "template" and I'll DM it.
```

Character count: ~1420. Post type: Value Bomb / Lead Magnet hybrid.

## Worked Example 2 — Case Study Bridge

```
[HOOK]
A [ICP: B2B consultant] went from [$X/mo] to [$3X/mo] in 11 weeks.
One variable changed. Here's the variable.

[LOOP]
Starting state: 
— Offer: hourly consulting, $[X]/hr, 1-off engagements
— Pipeline: referrals only
— Revenue: $[AMOUNT]/mo, heavily variable

The variable we changed: offer structure.

Replaced hourly consulting with a 90-day outcome-guaranteed engagement. Price: $[3-5X] flat. Same deliverable shape, different economics.

Week 1-2: rewrite the sales page around the 90-day outcome.
Week 3-4: run the new offer past 3 warm leads. Closed 2.
Week 5-8: adjust the onboarding to deliver to the outcome contract.
Week 9-11: ran the same outreach sequence. Closed [N] new engagements.

End state:
— Offer: 90-day outcome, $[3-5X] flat
— Pipeline: same sources, new structure
— Revenue: 3.1× prior month

The market was the same. The skills were the same. The client pool was the same. The container the work was sold inside changed — and that changed the price, the close rate, and the size.

[CTA]
If you're selling by the hour and want to see how this restructures, book a 15-min call: link in comments.
```

Character count: ~1580. Post type: Case Study + Offer-Tied.

## Worked Example 3 — Hot Take + Lead Magnet

```
[HOOK]
Stop sending "just checking in" follow-ups.
I tested [N] versions in [TIMEFRAME]. The worst variant outperformed "just checking in" by [X]×.

[LOOP]
"Just checking in" signals three things to the recipient:
— You don't have anything new to say
— You're measuring your activity, not their outcome
— You're running a script

Any of those kills reply rate.

The variants I tested:
1. Value add: shared a specific resource tied to what they asked about
2. Outcome reminder: referenced the specific outcome they said they wanted
3. Direct ask: "is this still a priority or should I close the file?"
4. Silence: didn't follow up at all, waited for them to re-engage

Variant 3 (direct ask) outperformed every other variant, including the value-add one.

Why: it respects the recipient's time and signals that you're not attached to a yes. Detachment signals quality. Attachment signals desperation.

Save this for your next follow-up sequence — build around direct asks, not check-ins.

[CTA]
I wrote the full 14-step follow-up sequence — direct asks, timing, what to send after a no. Comment "follow-up" and I'll DM it.
```

Character count: ~1440. Post type: Hot Take + Lead Magnet.

## Slot-Fill Rules

### The Hook slot
- Pull from `hooks/` swipe file (curiosity, specific-result, contrarian depending on post intent)
- Line 1 ≤ 140 chars visible before "see more"
- Line 2 widens the gap or deepens the stake

### The Loop slot
- Keep tension — every paragraph should earn the next
- Specificity over generality every time
- Visual break every 3-5 lines (blank line, bullet, bold)
- Use the creator's `phrases_to_use` 3+ times
- If teaching: named framework, numbered steps, bold key terms
- If story: 3-A structure (Arrival / Adversity / Aftermath)

### The CTA slot
- One action
- Specific: "comment 'X' and I'll DM 'Y'" beats "DM me if interested"
- Match the asset: if authority post → "follow for more"; if offer-tied → "book a call"; if lead magnet → "comment [word]"

## Format Choices

| Loop Format | When |
|---|---|
| Paragraph body | Story, authority, opinion |
| Bullet list | Listicle, framework, checklist |
| Numbered list | Steps, countdown, sequence |
| Q&A | FAQ, objection handling |
| Short scenes | Case study, day-in-life |
| Data/metrics | Specific-result, proof-forward |

---

## First Comment Plan

- 60-90 min after publish
- Drop a related resource link (in comment, not body) or a second insight
- Tag no one
- Reply to all comments within 2 hours of publish

---

## Common Failure Modes

- Hook too generic ("I learned something cool")
- Loop loses tension (becomes a list of disconnected points)
- CTA stacks three asks
- External link in body (70% reach reduction)
- Daily cadence (Dec 2025 algo penalty)
