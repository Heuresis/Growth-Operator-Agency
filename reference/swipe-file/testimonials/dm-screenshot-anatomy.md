# DM Screenshot Testimonial Anatomy

> **Anatomy of a compelling DM screenshot.** What makes one land vs flop. Collection protocol included.

---

## Why DM Screenshots Work

- Read as "unprompted" — reader infers the client wrote it unpressured
- Show real specificity (names, timestamps, platform artifacts)
- Low production cost = high trust perception
- Native to the channel where they're shared (IG story, LinkedIn, etc.)

---

## The 5 Elements Of A Compelling DM Screenshot

```
1. The sender's name / avatar (identifies the source)
2. The message text (the actual testimonial)
3. The timestamp (proves recency)
4. The context (what you sent them that prompted the reply)
5. The specific detail (a number, a name, a concrete moment)
```

If any element is missing, the screenshot loses weight.

---

## The Weight Hierarchy

Strongest DM testimonials, ranked:

### Tier 1 — Specific + Dated + Unprompted + Real-name
A DM that arrived 2 weeks after engagement ended, unprompted, with specific numbers, from a real named client.

Example structure:
```
[Sender: Real First Name L.]
[Time: 2 days ago]

"Hey — just wanted to follow up on what we talked about last month. 
Ran the [SPECIFIC MOVE] exactly as we discussed. Closed 3 [CLIENT TYPE] this week, 
at $[SPECIFIC AMOUNT] each. That's already ~2× my month's target. 
Thank you again, this was the best consulting money I've ever spent."
```

Why it lands: dated, specific, unprompted, names the action + outcome, and includes a dollar figure.

### Tier 2 — Specific + Prompted But Genuine
A DM reply to your question that's specific and honest.

Example:
```
[You asked:] "How's month 2 going?"

[Client reply:] "Actually pretty well — the [SPECIFIC MOVE] we agreed on in the last call 
produced [SPECIFIC OUTCOME]. I was honestly skeptical it would move the needle this fast 
but the data is clear. Need to keep pressure on [NEXT MOVE]."
```

Why it lands: prompted but not scripted, specific action + outcome, includes honest prior skepticism.

### Tier 3 — Emotional + Identity-Level
A DM that's less about metrics, more about identity or confidence shift.

Example:
```
"Wanted to send this before I forget: I had the first week where I didn't dread Sunday night 
since we started working together. Don't know the technical reason but something in my operation 
finally feels sustainable."
```

Why it lands: uses the client's own language, about identity not just numbers, specific to the week.

### Tier 4 — Early Wins / Small Specifics
A DM about a small, specific, early win.

Example:
```
"First outreach email using the new positioning just got a reply in 12 minutes. 
Under the old version I'd get maybe 1 reply per 40 sends. 
Small sample but directionally: this works."
```

Why it lands: honest caveat ("small sample"), specific numbers, specific change.

## Collection Protocol

How to ethically source DM testimonials:

1. **Don't fish** — don't send "say something nice" prompts.
2. **Follow up on real moments** — "how's month 2?" or "did the [SPECIFIC MOVE] land?"
3. **Ask for permission before screenshotting** — always.
4. **Offer to anonymize** — "would you prefer I blur your name?"
5. **Show them the screenshot before publishing** — approval step.

---

## Permission Ask Template

When the reply comes in that you want to use:

```
"Hey — this is great. Before I use it anywhere, want to check:

Would you be OK with me screenshotting this message and sharing it on [SPECIFIC CHANNEL]?

I can:
— Leave your name visible
— Blur your name but keep first name
— Blur everything identifying
— Extract the words only (no screenshot)

Let me know which you prefer. If you'd rather I don't use it at all, just say and I'll leave it off."
```

Result: client picks their comfort level. You ship ethically. They feel respected — which often produces more usable testimonials down the line.

## Design / Formatting

When using DM screenshots in content:

- Crop to show the full message but not other conversations
- Keep the platform chrome (helps with authenticity)
- Timestamp visible
- Sender name visible (or cleanly anonymized)
- Don't over-edit or add graphic overlays — keep native

---

## When NOT To Use DM Screenshots

- Content of the DM is about someone else (privacy risk)
- Timestamp shows it's 3+ years old (reads as stale)
- Sender is an unknown profile (reader trusts less)
- You faked / fabricated the DM (violation — zero tolerance)

---

## Combination Use

DM screenshots work best:
- As supporting proof under a full case study
- In carousel slides (1-2 screenshots per carousel)
- On IG story sequences (see `stories-instagram/lead-magnet-story.md` Day 6)
- On landing pages as social proof band

They work poorly:
- As the only testimonial on a sales page (reader wants longer form too)
- Stacked 10+ in a row (reads as spam)

---

## Common Failure Modes

- Screenshots without timestamps (feels staged)
- Sender names cropped weirdly (looks like hiding something)
- DMs that are too vague ("great work!")
- Over-stacking (10 screenshots in one post = noise)
- Publishing without permission (major violation)
- Faking screenshots (zero tolerance)
