# Listicle Thread — X/Twitter

> **Hook → N items → CTA.** 6-11 tweets. Save-optimized. Highest bookmark rate.

---

## The Skeleton

```
Tweet 1 — HOOK
"N [things] that [outcome]" format.
Specificity ≥ 8.
End with 🧵 or indicator.

Tweets 2 to N+1 — ITEMS
One item per tweet.
Item structure: bold lead + expansion + why it matters.

Final tweet — SAVE-TRIGGER + CTA
"Save this when [moment]."
One action.
```

Target: 5-10 items + 1 hook + 1 CTA = 7-12 tweets.

```

🧵

[Tweet 2]

[Tweet 4]

I thought hard work was the input. Revenue per hour is the output metric that actually matters.

[Tweet 5]

Designed the offer in isolation, then looked for who wanted it. Should have reversed: find audience → listen 6 weeks → design.

Pre-audience offers usually miss. Post-audience offers rarely do.

[Tweet 6]
5/ Keeping underperformers "to be kind."

Every drag on the team or client roster I kept "to be kind" cost 3× that kindness in operational tax.

The kindness was to everyone except me.

[Tweet 7]
6/ Over-optimizing the tech stack.

Spent [N] hours setting up [tools] I could have run on a spreadsheet.

Tools replace process. They don't create it.

[Tweet 8]
7/ Refusing to charge for discovery calls.

Free calls attract the cheapest prospects. Worth-winning clients rarely can't pay to talk.

Charging $[NUMBER] for a discovery call filtered the roster immediately.

[Tweet 9 — SAVE + CTA]

Want the full 30-item audit? DM me "audit" and I'll send it.
```

## Slot-Fill Rules

### Hook tweet
- Number in first 5 words ("7 mistakes...")
- Specificity ≥ 8
- Second line widens stake or previews insight

### Item tweets
- "N/" prefix for readability
- Bold lead (one sentence, the takeaway)
- 2-3 line expansion
- Parallel structure across all items

### Save + CTA tweet
- Explicit "save this when [moment]"
- One CTA
- Often tied to a Lead Magnet ("DM 'word' for full version")

---

## Parallel Structure Test

Every item should have the same shape:
- Item lead (one sentence)
- Expansion (2-3 lines)
- Why it matters / what to do

Breaking parallelism hurts save rate. Readers' brains pattern-match; broken patterns lose.

---

## Common Failure Modes

- Items aren't parallel
- Items cram two concepts per tweet
- Missing save-trigger close
- Generic framing (any creator could have written it)
- Thread goes to 13+ tweets
