# Educational Thread — X/Twitter

> **Thread (standard).** Hook → one idea per tweet → CTA. 7-12 tweets. Default authority format.

---

## The Skeleton

```
Tweet 1 — HOOK
Compress the thesis into 120-200 chars. 
End with 🧵 or ⬇️ or a number indicator (e.g., "7 ideas below:")

Tweets 2-N — BODY (one idea per tweet)
Each tweet = one concept, one paragraph-shaped chunk.
~180-280 chars per tweet.
Sequential: each tweet builds on the previous.

Final tweet — CTA
One action: Follow / DM / Link-in-reply.
```

Target: 7-12 tweets. Sweet spot for retention-per-tweet. 15+ tweets = steep drop-off.

## Slot-Fill Rules

### Hook tweet
- Specific (number / named entity / bold claim)
- Ends with 🧵 or ⬇️ or count indicator
- Promises something concrete
- Never "A thread 🧵" alone

### Body tweets
- One idea per tweet (no cramming)
- 180-280 chars each
- Sequential logic
- Save-optimized (readers should want to bookmark key tweets)

### Final CTA tweet
- One action
- Specific (DM "word" > "DM me if interested")
- Match thread intent (authority → follow; lead magnet → DM word; case study → book call)

---

## Voice-Match

3+ `phrases_to_use` across the thread. Banned vocabulary swept.

---

## Reply-to-Thread + Quote-Tweet

T+24-48h: quote-tweet with an additional takeaway (drops 1 new insight).
T+48-72h: quote-tweet with the hook repeated + a specific line from the thread.

Both moves compound reach ~2×.

---

## Common Failure Modes

- Hook too generic ("great thread coming")
- Two ideas per tweet (cramming)
- Thread runs past 15 tweets (steep drop-off)
- Final tweet stacks multiple CTAs
- No reply-to-thread planned (leaves 2× reach on the table)
