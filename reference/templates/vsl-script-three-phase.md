# VSL Script — 3-Phase Template (Alternate Structure)

> **Source skill:** `/build-vsl` (3-phase variant)
> **Output location:** `output/build-vsl/{YYYY-MM-DD}-vsl-3-phase.md`
> **Structure:** 3-phase (Hook-Story-Stack) — a shorter, tighter alternative to the 15-step
> **Runtime target:** 12-25 minutes spoken length
> **Use when:** most-aware or product-aware audience, retargeting creative, follow-up VSL after a webinar, cold traffic only if pain is acute.

Replace every `[FILL]` with real content. Do not ship with brackets remaining.

## Structure Overview

| Phase | Duration | Objective | Beliefs installed |
|---|---|---|---|
| Phase 1 — Open | 2-4 min | Hook + stakes + mechanism plant + credential | 1, 5, 7 |
| Phase 2 — Middle | 6-12 min | Problem redefined + 2 principles + proof | 2, 3, 6, 8 |
| Phase 3 — Close | 4-9 min | Offer + stack + guarantee + urgency + CTA | 4, 5, 6, 8 |

## Phase 1 — Open

### 1.1 Hook (0:00-0:20)

**Hook type:** [curiosity-gap / contrarian / big-number / identity call-out]

**Script:**
> [FILL — 20-40 words. Pattern interrupt. Must include one of: specific number, named entity, emotional word, unusual visual. No "Hey everyone." No "Let me tell you...".]

**Visual cue:** [FILL]

### 1.2 Promise + Stakes (0:20-1:00)

**Script:**
> In the next [N] minutes, I'm going to show you [FILL — specific outcome]. By the end you'll [FILL — capability]. If you close this and keep doing what you're doing, you'll [FILL — cost of inaction].

### 1.3 Name the Mechanism (1:00-1:45)

**Script:**
> The reason this hasn't worked for you isn't [common blame]. It's something I call [mechanism name]. Here's what it is in one sentence: [FILL].
>
> Once you see it, you can't un-see it. And once you apply it, [FILL — outcome shift].

### 1.4 Credential anchor (1:45-3:00)

**Script:**
> Quick context on why you should listen. I'm [FILL — creator name]. Over the last [N years] I've [FILL — credentialing sentence — not a CV read]. I've helped [N] [archetype] [FILL — specific outcome].
>
> Let me show you what this looks like.

## Phase 3 — Close

### 3.1 Transition (12:00-13:00)

**Script:**
> Now, could you figure this out on your own? Maybe. It took me [N years] to put it all together. Most [archetype] never finish the puzzle because [FILL — real barrier].
>
> I built a shortcut.

### 3.2 Offer Reveal (13:00-14:30)

**Script:**
> [Offer Name] is the [delivery format] that gives you [mechanism] end-to-end — in [timeframe]. It's built for [ICP archetype] who want [outcome] without [pain].
>
> Here's what's inside.

### 3.3 Stack (14:30-18:00)

**Core components:**
> - [Component 1] — [one line] — worth $[N]
> - [Component 2] — worth $[N]
> - [Component 3] — worth $[N]
> - [Component 4] — worth $[N]
> - [Component 5] — worth $[N]

**Bonuses (2nd-best first, best last):**
> - [Bonus 1] — counters [objection] — worth $[N]
> - [Bonus 2] — counters [objection] — worth $[N]
> - [Bonus 3] — counters [objection] — worth $[N]
> - [Bonus 4] — counters [objection] — worth $[N]
> - [Bonus 5] (climax) — counters [objection] — worth $[N]

**Total stack value:** $[SUM]

**Price reveal:**
> If you built this piece by piece, it would cost $[anchor]. If you hired it done, $[higher anchor]. Your investment is $[primary price].

### 3.4 Guarantee (18:00-19:30)

**Script:**
> My promise to you: [FILL — guarantee copy from Offer Doc Section 8.5].
>
> The only way you lose is if you don't [reasonable condition].

### 3.5 Urgency (19:30-20:30)

**Real scarcity mechanism (pick one):**
- Cohort start date
- Seat cap
- Bonus expiry
- Scheduled price increase

**Script:**
> [FILL — specific real scarcity mechanism]. After that, [what changes].

**NEVER fabricate.** If no real scarcity, skip this step.

### 3.6 CTA + Close (20:30-end)

**Script:**
> Here's what to do next. Click the button below. Fill in [next-step action]. You'll [FILL — what happens next].
>
> I'll see you inside [Offer Name].

**Final frame:** CTA + URL + real-scarcity reminder.

## Appendix B — When to Use 3-Phase vs 15-Step

| Condition | Use 15-step | Use 3-phase |
|---|---|---|
| Cold traffic, problem-aware | ✅ | ❌ |
| Cold traffic, solution-aware | ✅ | ⚠ if pain is acute |
| Retargeting / warm | ⚠ long | ✅ |
| Post-webinar follow-up | ❌ | ✅ |
| Most-aware list | ❌ | ✅ |
| Sophistication: Naive / Aware | ✅ | ⚠ |
| Sophistication: Skeptical / Exhausted | ❌ too long | ✅ tight + credible |

## Appendix C — Performance Targets

| Metric | Target |
|---|---|
| View-through at 60 sec | >= 55% |
| View-through at 50% mark | >= 35% |
| View-through to CTA | >= 20% |
| CTA click rate | >= 30% |

## Appendix D — Filming Notes

- Direct-to-camera for Phases 1 and 3.
