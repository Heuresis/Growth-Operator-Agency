## The 5 VSM Subsystems

| System | Role | In a business |
|---|---|---|
| **S1 — Operations** | Produce/consume signals (core work) | Production, fulfillment, delivery |
| **S2 — Coordination** | Prevent S1 conflicts | Scheduling, rules, priorities |
| **S3 — Control** | Manage flow + resources (3* = audit) | Ops management, accountability |
| **S4 — Intelligence** | Scan external environment | Market research, competitive intel |
| **S5 — Policy** | Define identity + encoding standards | Vision, values, brand, strategic direction |

**Recursive rule:** each operational subsystem must itself contain all 5.

## Growth Optimal System 7-Department Mapping

| Department | Primary VSM Role | Why |
|---|---|---|
| **Foundations** | **S5 Policy** | ICP + Offer + Positioning = identity definition. Sets encoding standards for every downstream department. |
| **Marketing** | **Transmission Layer** (S1/S4 bridge) | Transmits S5's identity outward; scans external (trends, competitors) back inward |

## How the VSM Check Validates Growth Optimal System

### S5 check — Is identity defined?
Foundations compartment (Creator Identity Matrix + Offer + Positioning) must be ≥ 70%. Below that = no S5 = no viable system.

### S2 check — Is there coordination?
Nurture sequences + content-to-conversion bridge must exist. Without them, stages conflict (e.g., cold traffic lands on offer-page = bad-fit).

### S3 check — Is there control?
Launch plans + operations SOPs must exist. Without them, execution drift = no control.

### S4 check — Is intelligence captured?
`/research`, `/competitor-intel`, and Partnerships scan the environment. Without them, blind to market shifts.

### S1 check — Is operational execution happening?
Scale SOPs + team structure + revenue reports = S1. Without them, no output.

## The Recursive Principle

Each department is itself a viable system:

**Foundations (S5 at business level) internally has its own S1-S5:**
- S1 = `/research` output, `/build-icp` output, `/design-offer` output (operational artifacts)
- S2 = sequential dependency chain (Research → ICP → Positioning → Offer)
- S3 = CONTEXT-THRESHOLDS.md gates (controls what ships when)
- S4 = market scanning during `/research`
- S5 = INVARIANTS.md applied to Foundations (truth gate, 40/40/20, etc.)

Same recursion applies to every department.

## The 3* Audit Layer

viable-systems-theory 3* = audit function separate from S3:
- S3 manages resources (normal operations control)
- S3* audits S1 directly (catches what S3 misses)

**In Growth Optimal System:**
- S3 = Launch-Lead + Scale-Lead (run operations)
- S3* = Blind Output Test + Triple-Layer Verification + /retention-check (audit operations)

Without S3*, S3 + S1 can drift into incestuous optimization. The audit prevents that.

## Algedonic Channel (escalation bypass)

the algedonic = emergency pain/pleasure signal that bypasses normal channels and reaches S5 directly.

**In Growth Optimal System:** INVARIANTS.md violations + refund spikes + brand-safety issues = algedonic. Escalate directly to creator + growth-ceo agent, bypassing normal review cadence.

## Applied Diagnostic

When a Growth Optimal System instance has problems, VSM-diagnose:

| Symptom | Likely VSM gap |
|---|---|
| Generic output everywhere | S5 weak (identity undefined — Foundations low completeness) |
| Content + ads + VSL contradict each other | S2 weak (coordination missing) |
| Execution drift during launch | S3 weak (launch control gap) |
| Surprised by market shift | S4 weak (no environmental scan) |
| No revenue despite plans | S1 weak (operational execution gap) |
